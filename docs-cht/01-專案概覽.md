# OpenClaw 專案概覽

## 什麼是 OpenClaw？

OpenClaw 是一個強大的多頻道 AI 閘道器平台，支援多種訊息通道（WhatsApp、Telegram、Discord、Slack、Signal、iMessage 等），讓使用者能夠透過熟悉的聊天介面與 AI 互動。

### 核心特色

1. **多通道支援** - 連接超過 30 種訊息平台
2. **AI Agent** - 基於 Claude、GPT、Mistral 等大型語言模型的智慧代理
3. **工具整合** - 程式碼執行、瀏覽器控制、檔案操作等
4. **跨平台** - 支援 macOS、iOS、Android、Linux、Windows
5. **可擴充** - 完整的插件系統（37 個擴充套件）

## 版本資訊

- **目前版本**: `2026.3.2`
- **Node.js 需求**: `>=22.12.0`
- **套件管理器**: `pnpm@10.23.0`

## 主要功能

### 1. 通訊頻道
- **內建頻道**: WhatsApp、Telegram、Discord、Slack、Signal、iMessage、LINE、Google Chat
- **擴充頻道**: Matrix、MS Teams、Zalo、Mattermost、Nostr、Twitch、飛書、BlueBubbles、Synology Chat、Nextcloud Talk、IRC、Tlon 等

### 2. AI 功能
- **對話式 AI**: 自然語言互動
- **程式碼助手**: 程式碼生成、除錯、解釋
- **工具呼叫**: 檔案操作、網頁瀏覽、Shell 執行
- **多模態**: 圖片、音訊、文件理解
- **PDF 分析工具**: 原生 Anthropic/Google 提供者支援，非原生模型自動降級提取
- **子代理 (Subagents)**: 多 Agent 並行協作、跨代理生成、結構化完成事件、內聯附件
- **記憶搜尋**: FTS 多語言支援（含日、韓、阿拉伯語等）
- **ACP/ACPX 協議**: Agent Client Protocol 支援（ACPX 串流，dispatch 預設啟用）
- **Diffs 工具**: 唯讀差異渲染（before/after 或 unified patch，支援 PDF 輸出）
- **Thinking 自適應**: Claude 4.6 預設 `adaptive` 思考模式
- **音頻轉錄回聲**: `echoTranscript` 選項在代理處理前向聊天發送轉錄確認
- **Hooks 生命週期**: `message:transcribed`、`message:preprocessed`、`message:sent`（含 `isGroup`/`groupId`）

### 3. Gateway 伺服器
- **WebSocket RPC**: 即時雙向通訊
- **多客戶端**: Web UI、CLI、行動應用同時連接
- **會話管理**: 對話歷史、上下文壓縮（compaction）、自動清理維護
- **健康檢查端點**: `/health`、`/healthz`（liveness）、`/ready`、`/readyz`（readiness），適用於 Docker/Kubernetes
- **安全掃描**: 技能/插件程式碼安全分析
- **Secrets 管理**: 秘密引用（SecretRef）與執行時解析
- **Cron 排程**: 定時任務系統（支援輕量 bootstrap 模式、失敗目標路由）
- **配置驗證**: `openclaw config validate` 在啟動前驗證設定檔
- **模型熱重載**: 心跳偵測 `models.*` 配置變更並自動重載，無需重啟 Gateway
- **Plugin HTTP 路由**: 顯式外掛路由優先於 Control UI SPA catch-all

### 4. 原生應用
- **macOS 選單列應用**: Gateway 管理、聊天介面、Canvas
- **iOS 應用**: 行動節點、相機、語音、setup-code 配對、Apple Watch
- **Android 應用**: 行動節點、SMS、螢幕錄製、通知操作（open/dismiss/reply）、相機列表、裝置健康、聯絡人/行事曆/動態感測器

### 5. 安全與維運
- **技能/插件安全掃描器**: 偵測危險執行、加密挖礦、程式碼混淆等
- **會話自動維護**: 過期清理、大小限制、檔案輪轉
- **命令授權分離**: `commands.allowFrom` 細粒度控制
- **設備配對管理**: `device-pair` 擴充套件
- **配置值脫敏**: `config get` 輸出自動遮蔽敏感值
- **執行審批**: exec 工具安全策略
- **沙箱繼承**: `sessions_spawn` 沙箱邊界不允許降級
- **TOCTOU 防護**: 工作區安全寫入防護 symlink 競爭條件
- **節點元資料策略**: Unicode 混淆防護、保守預設允許清單
- **Shell 環境標記**: `OPENCLAW_SHELL` 跨 shell 執行環境注入
- **Webhook Auth-Before-Body**: Webhook 先驗證身份再讀取 body，防止慢速 DoS
- **Skills Symlink 審計**: `security audit` 偵測 workspace 外部 symlink 逃脫
- **Prompt Spoofing 防護**: 中和入站訊息中的偽造 `[System Message]` 標記
- **Skills Archive 提取安全**: tar.gz/tar.bz2 統一安全檢查，強制壓縮大小限制

## 專案歷史

OpenClaw 源自對「隨處可用的 AI 助手」願景的追求。專案最初聚焦於 WhatsApp 整合（使用 Baileys 協議），後逐步擴展到多通道支援，並發展出完整的 Agent 框架和插件生態系。
