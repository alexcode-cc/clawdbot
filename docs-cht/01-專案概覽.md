# OpenClaw 專案概覽

## 什麼是 OpenClaw？

OpenClaw 是一個強大的多頻道 AI 閘道器平台，支援多種訊息通道（WhatsApp、Telegram、Discord、Slack、Signal、iMessage 等），讓使用者能夠透過熟悉的聊天介面與 AI 互動。

### 核心特色

1. **多通道支援** - 連接超過 30 種訊息平台
2. **AI Agent** - 基於 Claude、GPT、Mistral 等大型語言模型的智慧代理
3. **工具整合** - 程式碼執行、瀏覽器控制、檔案操作等
4. **跨平台** - 支援 macOS、iOS、Android、Linux、Windows
5. **可擴充** - 完整的插件系統（43 個擴充套件）
6. **Fast Mode** - Anthropic / OpenAI 快速模式切換

## 版本資訊

- **目前版本**: `2026.3.24`
- **Node.js 需求**: `>=22.16.0`
- **套件管理器**: `pnpm@10.23.0`

## 主要功能

### 1. 通訊頻道
- **內建頻道**: WhatsApp、Telegram、Discord、Slack、Signal、iMessage、LINE、Google Chat
- **擴充頻道**: Matrix、MS Teams、Zalo、Mattermost、Nostr、Twitch、飛書、BlueBubbles、Synology Chat（含設定精靈）、Nextcloud Talk、IRC、Tlon 等

### 2. AI 功能
- **對話式 AI**: 自然語言互動
- **程式碼助手**: 程式碼生成、除錯、解釋
- **工具呼叫**: 檔案操作、網頁瀏覽、Shell 執行
- **多模態**: 圖片、音訊、文件理解
- **PDF 分析工具**: 原生 Anthropic/Google 提供者支援，非原生模型自動降級提取
- **子代理 (Subagents)**: 多 Agent 並行協作、跨代理生成、結構化完成事件、內聯附件
- **記憶搜尋**: FTS 多語言支援（含日、韓、阿拉伯語等），Ollama 嵌入向量支援
- **ACP/ACPX 協議**: Agent Client Protocol 支援（ACPX 串流，dispatch 預設啟用）
- **Diffs 工具**: 唯讀差異渲染（before/after 或 unified patch，支援 PDF 輸出）
- **Web 搜尋**: Perplexity、Brave（預設啟用）、DuckDuckGo（experimental）、Exa、Tavily、Gemini、Grok、Kimi — 全部透過 Plugin Capability 系統運作
- **Context Engine 外掛**: 自訂 context management 策略（`kind: "context-engine"` manifest）
- **工具迴圈偵測**: 可選的防重複工具呼叫護欄（`tools.loopDetection`）
- **工具結果截斷**: head+tail 截斷保留重要尾部診斷資訊
- **Thinking 自適應**: Claude 4.6 預設 `adaptive` 思考模式
- **音頻轉錄回聲**: `echoTranscript` 選項在代理處理前向聊天發送轉錄確認
- **Hooks 生命週期**: `message:transcribed`、`message:preprocessed`、`message:sent`（含 `isGroup`/`groupId`）、`compaction:start`/`compaction:end`
- **Compaction 模型覆寫**: 壓縮可使用獨立模型（`compaction.model` 配置）
- **Compaction Safeguard**: 品質審計重試、結構化摘要標題、preserve/quality 設定
- **本地備份 CLI**: `openclaw backup create`/`verify` 本地配置備份
- **西班牙語支援**: 新增 Spanish locale
- **UTC 時間併行**: 系統提示詞同時顯示本地和 UTC 時間
- **Slim Docker Image**: 精簡版 runtime image 選項
- **Fast Mode**: Anthropic / OpenAI session-level 快速模式（`/fast` 切換）
- **Dashboard v2**: 全新翻新的 Control UI dashboard（模組化 views、command palette、行動版 tabs）
- **Chrome DevTools MCP**: Chrome existing-session attach 模式（signed-in live browser）
- **瀏覽器批次操作**: batched actions、selector targeting、delayed clicks
- **Sessions Yield**: `sessions_yield` 工具（cooperative turn-ending）
- **Kubernetes 安裝**: K8s raw manifests、Kind 設定、部署指南
- **Provider Plugin 架構**: Ollama/vLLM/SGLang 遷移至 provider-plugin 架構
- **Slack 互動式回覆**: Block Kit 訊息、opt-in 按鈕和選擇器回覆
- **Docker 時區**: `OPENCLAW_TZ` 環境變數設定容器時區
- **Gemini Embedding 2**: gemini-embedding-2-preview 記憶嵌入支援
- **多模態記憶**: 圖片和音訊索引支援
- **Mattermost replyToMode**: 回覆串接行為控制（off/first/all）
- **Plugin Capability 系統**: Web Search、Image Generation、TTS/Speech、Media Understanding 完整遷入插件架構
- **ClawHub 插件市場**: 插件安裝預設從 ClawHub 取得，支援 `/install` 快捷安裝
- **Image Generation**: `image_generate` 工具（Google、fal.ai providers）
- **TTS 插件化**: Speech providers（Deepgram、Groq、Microsoft）遷入插件，記憶體內語音合成
- **Anthropic Vertex**: Claude via GCP Vertex AI 提供者
- **xAI (Grok)**: 完整整合 + fast mode
- **GitHub Copilot**: 動態 model ID 解析
- **Context Engine 增強**: transcript 維護、assemble() prompt/modelId 傳遞、JSONL 截斷
- **Compaction 通知**: 壓縮開始和完成時通知使用者
- **Nostr 頻道**: 新增 Nostr 協議頻道支援
- **OpenShell 沙箱**: 遠端 SSH 沙箱後端
- **Synology Chat 精靈**: 引導式設定精靈
- **Telegram 增強**: DM 主題自動重命名、自訂 API 端點、靜默錯誤回覆
- **DuckDuckGo / Exa / Tavily**: 三個新 bundled web search plugins
- **Outbound 插件化**: delivery routing 遷至 channel plugins
- **Reply Lazy-Load**: 12+ 項 lazy-load 優化降低啟動和回覆延遲
- **Xiaomi MiMo V2**: MiMo V2 Pro 和 Omni 模型
- **Chutes 擴充**: 新增 bundled Chutes extension
- **OpenAI 相容端點**: Gateway 新增 `/v1/models`、`/v1/embeddings`（Agent-first）
- **MiniMax Image Gen**: MiniMax 新增 image generation provider（M2.7）
- **Discord autoThreadName**: LLM 自動生成 thread 名稱策略
- **MS Teams 編輯/刪除**: 訊息編輯和刪除、結構化引用上下文
- **Container CLI**: 支援定向至容器化 OpenClaw 實例
- **/tools 視圖**: 對話中查看工具 runtime 可用性狀態
- **Control UI 打磨**: Skills 翻新、Markdown 預覽、workspace 瀏覽、macOS config tree

### 3. Gateway 伺服器
- **WebSocket RPC**: 即時雙向通訊
- **多客戶端**: Web UI、CLI、行動應用同時連接
- **會話管理**: 對話歷史、上下文壓縮（compaction）、自動清理維護
- **健康檢查端點**: `/health`、`/healthz`（liveness）、`/ready`、`/readyz`（readiness），適用於 Docker/Kubernetes
- **安全掃描**: 技能/插件程式碼安全分析
- **Secrets 管理**: SecretRef 支援 64 個憑證目標，涵蓋完整使用者提供的憑證表面
- **Cron 排程**: 定時任務系統（支援輕量 bootstrap 模式、失敗目標路由）
- **配置驗證**: `openclaw config validate` 在啟動前驗證設定檔
- **模型熱重載**: 心跳偵測 `models.*` 配置變更並自動重載，無需重啟 Gateway
- **Plugin HTTP 路由**: 顯式外掛路由優先於 Control UI SPA catch-all
- **Gateway Auth SecretRef**: `gateway.auth.token` 支援 SecretRef 引用
- **Gateway 密碼檔案**: `--password-file` 從檔案讀取密碼
- **Channel Readiness Probes**: 透過頻道連線狀態判斷 gateway 就緒
- **HTTP 安全標頭**: 預設 `Permissions-Policy: camera=(), microphone=(), geolocation=()`
- **Agent 發現**: 磁碟掃描 agent IDs 納入 `listConfiguredAgentIds`
- **Dashboard v2**: 全新翻新的 dashboard（modular overview/chat/config/agent/session views）
- **Gateway --require-rpc**: `status --require-rpc` 強制 RPC 探測失敗
- **用戶端請求限時**: 限時並清理懸掛的 RPC calls
- **Gateway runtime version**: 狀態中暴露 runtime 版本
- **會話 reset 路由保留**: 保留 lastAccountId/lastThreadId 跨 reset
- **Kubernetes 支援**: K8s manifests、Kind 設定、部署文件
- **Gateway SIGTERM**: 強化 SIGTERM 關閉行為
- **Doctor Plugin 摘要**: workspace status 中新增 bundle plugin capability 摘要
- **Doctor Plugin 清理**: 清理過期的 plugin allowlist 和 entry refs
- **Talk Speak RPC**: 新增 talk speak RPC 方法
- **Webchat 圖片持久化**: inbound 圖片持久化至磁碟

### 4. 原生應用
- **macOS 選單列應用**: Gateway 管理、聊天介面、Canvas
- **iOS 應用**: 行動節點、相機、語音、setup-code 配對、Apple Watch、Live Activity 連線狀態、首次啟動 Welcome Pager、Home Canvas 工具列、APNs Relay Gateway、本地 Beta 發布流程
- **Android 應用**: 行動節點（套件名 `ai.openclaw.app`）、SMS、螢幕錄製、通知操作（open/dismiss/reply）、相機列表、裝置健康、聯絡人/行事曆/動態感測器、全新 Onboarding UI、聊天設定重設計、Google Code Scanner

### 5. 安全與維運
- **技能/插件安全掃描器**: 偵測危險執行、加密挖礦、程式碼混淆等
- **會話自動維護**: 過期清理、大小限制、檔案輪轉
- **命令授權分離**: `commands.allowFrom` 細粒度控制
- **設備配對管理**: `device-pair` 擴充套件
- **配置值脫敏**: `config get` 輸出自動遮蔽敏感值
- **執行審批**: exec 工具安全策略
- **沙箱繼承**: `sessions_spawn` 沙箱邊界不允許降級
- **TOCTOU 防護**: 工作區安全寫入防護 symlink 競爭條件
- **節點元資料策略**: Unicode 混淆防護、保守預設允許清單
- **Shell 環境標記**: `OPENCLAW_SHELL` 跨 shell 執行環境注入
- **Webhook Auth-Before-Body**: Webhook 先驗證身份再讀取 body，防止慢速 DoS
- **Skills Symlink 審計**: `security audit` 偵測 workspace 外部 symlink 逃脫
- **Prompt Spoofing 防護**: 中和入站訊息中的偽造 `[System Message]` 標記
- **Skills Archive 提取安全**: tar.gz/tar.bz2 統一安全檢查，強制壓縮大小限制
- **Sandbox fs-bridge 強化**: readfile handle 固定、mkdirp 路徑錨定、破壞性操作錨定
- **SSRF 重定向檢查**: 瀏覽器/跨域重定向時移除自訂 auth headers
- **Zip 提取安全**: 強化 path traversal 防護
- **MIME Spoofing 防護**: 拒絕偽造的 input_image MIME payload
- **Config Log Injection 防護**: 配置驗證輸出控制字元消毒
- **工作區插件信任**: 停用隱含的工作區插件自動載入，需明確信任決定（GHSA-99qw-6mr3-36qr）
- **裝置配對 Token**: 配對 setup codes 切換為短期單次使用 bootstrap tokens
- **iMessage SCP 防護**: 拒絕不安全的遠端附件路徑，防止 shell metacharacter 注入
- **Telegram Webhook**: 先驗證 secret 再讀取 body
- **Exec Approval 全面強化**: pnpm 解包、Perl -M/-I 封鎖、PowerShell -File 辨識、env dispatch 解包、shell line continuation、skill trust 路徑綁定
- **SecretRef 遍歷拒絕**: 跨 schema/runtime/gateway 的 exec SecretRef 遍歷 ID 拒絕
- **Feishu Webhook 加密**: 要求 webhook encrypt key
- **外部內容標記**: 剝離零寬度和 soft-hyphen 標記分割字元
- **蒙古語選擇器**: exec 混淆偵測器剝離蒙古語選擇器
- **子代理控制邊界**: 限制 leaf subagent 控制範圍
- **WebSocket Origin**: 強制瀏覽器 origin 檢查（不受 proxy headers 影響）
- **Canvas Route Auth**: Canvas 路由要求身份驗證、agent session reset 需要 admin
- **媒體路徑安全**: 封鎖遠端主機 file:// URL、強化次要本地路徑邊界
- **Plugin 來源限制**: 限制遠端 marketplace plugin 來源
- **Gateway 安全**: 未解析 discovery endpoints fail closed、internal command 持久化 gating
- **診斷資料遮蔽**: cache-trace 診斷輸出遮蔽認證資料
- **Android Canvas Bridge**: 限制至受信任頁面
- **Nostr DM 政策**: 解密前強制 inbound DM 政策
- **Synology Chat**: 收緊 reply name matching 和 shared webhook fail closed
- **SecOps 所有權**: CODEOWNERS 為敏感路徑指定 secops 團隊
- **Exec Approval 持續強化**: shell-wrapper argv 匹配收緊、dispatch wrapper 統一、不可見字元轉義、jq safe-bin policy、host env override 強化
- **Sandbox 修復（2026.3.24+）**: alsoAllow 遵從、媒體根繞過封閉、session_status 可見性強制
- **Channel Auth 安全（2026.3.24+）**: prototype-chain 污染防護、控制字元注入防護
- **Aisle 審計修復（2026.3.24+）**: skill 安裝器驗證、terminal 消毒、URL scheme 白名單
- **CSP Inline Hashes（2026.3.24+）**: Control UI 支援 inline script hashes

## 專案歷史

OpenClaw 源自對「隨處可用的 AI 助手」願景的追求。專案最初聚焦於 WhatsApp 整合（使用 Baileys 協議），後逐步擴展到多通道支援，並發展出完整的 Agent 框架和插件生態系。
