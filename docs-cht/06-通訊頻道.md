# 通訊頻道

## 概述

OpenClaw 支援超過 30 種訊息平台，讓使用者能透過熟悉的聊天介面與 AI 互動。頻道分為內建頻道和擴充頻道兩類。

## 內建頻道

### WhatsApp (Baileys)

**技術**: Baileys（WhatsApp Web 協議）

**特色**:

- 掃描 QR Code 登入
- 支援私訊和群組
- 媒體訊息（圖片、音訊、文件）
- 即時狀態（輸入中、已讀）

**配置**:

```json5
{
  channels: {
    whatsapp: {
      enabled: true,
      allowlist: ["+1234567890"],
    },
  },
}
```

**程式碼位置**: `src/web/`（含 `inbound/`, `auto-reply/` 子目錄）

### Telegram

**技術**: grammY（Telegram Bot API）

**特色**:

- Bot API（需要 Bot Token）
- 支援私訊和群組
- 豐富的訊息格式（Markdown、按鈕）
- Webhook 或 Polling 模式
- **串流預設 `partial`**（新安裝預設開啟即時預覽串流）
- **DM 私聊串流**（使用 `sendMessageDraft`，reasoning/answer 通道分離）
- **`disableAudioPreflight`**：跳過群組/topic 入站語音訊息的提及偵測前飛轉錄
- **Per-topic agentId 路由**：forum groups 和 DM topics 可設定個別 `agentId` 覆寫，路由到專屬 agent 並隔離 session
- Per-DM topic 配置（`direct` + topic allowlists、`dmPolicy`、`skills`、`systemPrompt`、`requireTopic`）
- DM topic 會話隔離（`<chatId>:<threadId>` scoped session key）
- 多帳號群組隔離（groups config 不再跨帳號洩漏）
- 多帳號預設路由：2+ 帳號未設 default 時僅發出警告，`openclaw doctor` 顯示缺失/無效預設帳號
- Plugin outbound hooks：Telegram 回覆路徑執行 `message_sending` + `message_sent` hooks
- 回覆媒體上下文（replied media files in inbound context）
- 語音 caption overflow 自動降級
- `replyToMode: "first"` chunking 支援
- 群組 allowlist 順序修正（chat allowlist 優先於 sender allowlist）
- 長模型按鈕 compact callback fallback（防止 Telegram 64 bytes callback 截斷）
- forum system messages 不觸發隱式提及偵測
- 保留入站媒體原始 `file_name` 元資料

**配置**:

```json5
{
  channels: {
    telegram: {
      enabled: true,
      token: "BOT_TOKEN",
      allowlist: ["@username", "123456789"],
    },
  },
}
```

**程式碼位置**: `src/telegram/`

### Discord

**技術**: Carbon

**特色**:

- Bot API（需要 Bot Token）
- 支援 DM 和伺服器頻道
- 斜線命令
- 訊息附件
- **`allowBots: "mentions"`**：僅接受提及機器人的 bot 訊息
- 串流預覽配置（`channels.discord.streaming`）
- Thread 綁定生命週期管理（`idleHours`/`maxAgeHours` 取代固定 TTL）
- `/session idle` 和 `/session max-age` 指令
- Components v2 支援（`cid` payload）
- Forum thread 標籤（`appliedTags`）
- 每帳號 `ackReactionScope` 覆寫
- 多帳號 `eventQueue.listenerTimeout` 配置
- Thread session 重置：thread 封存時重置 session，重新開啟時從新 session 開始
- 預設線上狀態：無自訂 presence 時自動發送 online presence
- Mention 處理：id-based 格式化 + cache 改寫、`ignoreOtherMentions` 設定
- Media SSRF allowlist：允許 Discord CDN 主機名（含萬用字元）
- Slash 命令路由：攔截文字 slash 命令、註冊外掛命令為原生、空 slash 執行 fallback
- Voice 解碼降級：使用 opusscript 取代原生 Opus 依賴
- Auto presence 健康信號：執行時可用性驅動的 presence 更新 + 連線狀態報告
- Agent media roots：`mediaLocalRoots` 透過 Discord reply delivery 傳遞

**配置**:

```json5
{
  channels: {
    discord: {
      enabled: true,
      token: "BOT_TOKEN",
      allowlist: ["user_id", "server_id"],
    },
  },
}
```

**程式碼位置**: `src/discord/`

### Slack

**技術**: @slack/bolt

**特色**:

- Slack App（需要 OAuth Token）
- 支援 DM 和頻道
- 互動式元件
- Webhook 模式
- **`typingReaction`**：Socket Mode DM 可透過反應表情顯示處理狀態（Slack 原生 assistant typing 不可用時的替代方案）
- **`replyToMode=off` session 路由**：top-level 頻道訊息共享單一 session
- **inbound debounce 路由**：按 message timestamp 隔離 debounce key，防止 cross-thread 衝突
- **thread context 優化**：現有 session 跳過 thread 起始/歷史文字注入，減少 token 浪費
- **Bolt 4.6 相容**：移除 `message.channels`/`message.groups` 事件處理器，改用 `channel_type` 統一 handler
- **mrkdwn 正規化**：串流和預覽路徑進行 mrkdwn 轉換

**配置**:

```json5
{
  channels: {
    slack: {
      enabled: true,
      token: "xoxb-...",
      signingSecret: "...",
      appToken: "xapp-...",
    },
  },
}
```

**程式碼位置**: `src/slack/`（含 `http/` 子目錄）

### Signal

**技術**: Signal 協議

**特色**:

- 端對端加密
- 私訊支援
- 需要 Signal CLI 或 signal-cli-rest-api

**配置**:

```json5
{
  channels: {
    signal: {
      enabled: true,
      number: "+1234567890",
    },
  },
}
```

**程式碼位置**: `src/signal/`

### iMessage

**技術**: imsg CLI（macOS 限定）

**特色**:

- macOS 原生整合
- 私訊和群組
- 需要 Full Disk Access

**配置**:

```json5
{
  channels: {
    imessage: {
      enabled: true,
      allowlist: ["+1234567890", "user@icloud.com"],
    },
  },
}
```

**程式碼位置**: `src/imessage/`

### LINE

**技術**: LINE Messaging API

**特色**:

- LINE Messaging API
- DM/群組/聊天室支援
- Webhook Auth-Before-Body 安全模式
- 嚴格 auth 邊界語意（配對 store、DM/群組 allowlist 分離、fail-closed）
- 重播/重複控制（in-flight reservation + post-success dedupe）
- 檔案媒體下載修復 + M4A 音訊分類
- 群組/聊天室 peer 路由 + 命令授權上下文傳播
- Status 探測快取（snapshot/config state）
- Webhook HEAD 探測相容

**程式碼位置**: `src/line/`

## 訊息路由

### Allowlist 機制

```json5
{
  channels: {
    telegram: {
      allowlist: ["@user1", "@user2", "group_id"],
    },
  },
}
```

Discord allowlist 名稱解析已規範化，確保一致的比對行為。Discord 新增 debug 日誌顯示 guild/channel allowlist 被丟棄的訊息。

### 多帳號預設路由

頻道支援可選的 `channels.<channel>.defaultAccount` 預設帳號選擇，當 `accountId` 未指定時路由到明確配置的帳號，而非依賴隱式排序。

### 串流預覽配置

統一的串流配置路徑：

```json5
{
  channels: {
    "<channel>": {
      streaming: {
        /* 串流預覽設定 */
      },
    },
  },
}
```

### DM 範圍

CLI 本地 onboarding 預設 `session.dmScope` 為 `per-channel-peer`。

## 開發新頻道

### 1. 建立擴充套件目錄

```
extensions/my-channel/
├── openclaw.plugin.json
├── index.ts
├── package.json
└── src/
    ├── channel.ts
    ├── monitor.ts
    ├── send.ts
    └── runtime.ts
```

### 2. 定義插件元資料

```json
// openclaw.plugin.json
{
  "id": "my-channel",
  "channels": ["my-channel"],
  "configSchema": {
    "type": "object",
    "properties": {
      "apiKey": { "type": "string" },
      "allowlist": { "type": "array", "items": { "type": "string" } }
    },
    "required": ["apiKey"]
  }
}
```

### 3. 實作頻道插件

```typescript
// src/channel.ts
import type { ChannelPlugin, MessageHandler } from "openclaw/plugin-sdk";

export const channelPlugin: ChannelPlugin = {
  id: "my-channel",
  name: "My Channel",

  async initialize(config) {
    /* 初始化 SDK */
  },
  async startMonitor(handler: MessageHandler) {
    client.on("message", (msg) => {
      handler({
        id: msg.id,
        channelId: "my-channel",
        chatId: msg.chatId,
        senderId: msg.sender,
        content: msg.text,
        timestamp: new Date(),
      });
    });
  },
  async sendMessage(chatId, message) {
    await client.send(chatId, message.content);
  },
  async getStatus() {
    return { connected: client.isConnected() };
  },
};
```

### 4. 註冊插件

```typescript
// index.ts
import type { OpenClawPluginApi } from "openclaw/plugin-sdk";
import { channelPlugin } from "./src/channel.js";

const plugin = {
  id: "my-channel",
  name: "My Channel",
  register(api: OpenClawPluginApi) {
    api.registerChannel({ plugin: channelPlugin });
  },
};

export default plugin;
```

### 5. 更新標籤

新增頻道/擴充套件時，需更新 `.github/labeler.yml` 並建立對應的 GitHub 標籤。

## WhatsApp 更新

- 入站 `fromMe` 上下文傳播，agent 可區分 owner 自發訊息（標記 `(self)`）
- 主 DM 路由改為單一 owner

## Synology Chat

新增/更新的 Synology Chat 整合功能：

- **Webhook 相容性**：接受 JSON 和別名 payload 欄位，支援 body/query/header token 來源，以 `204` ACK 防止 "Processing..." 狀態
- **Body read 邊界**：強制 size + timeout bound，防止慢速 DoS
- **回覆交付**：解析 webhook username → Chat API `user_id`
- **啟動循環修復**：`startAccount` 等到 abort 防止 webhook 路由重啟循環

## 飛書 (Feishu) 進階功能

### Docx 表格與上傳

```json5
{
  channels: {
    feishu: {
      tools: {
        doc: true, // feishu_doc 工具
        chat: true, // feishu_chat 工具
      },
    },
  },
}
```

- `feishu_doc` 操作：`create_table`、`write_table_cells`、`create_table_with_values`、`upload_image`、`upload_file`
- `feishu_chat` 操作：聊天資訊與成員查詢
- Reactions 處理：`im.message.reaction.created_v1` 事件
- 群組會話範圍：`group`、`group_sender`、`group_topic`、`group_topic_sender`
- `replyInThread` 配置（`disabled | enabled`）
- 多帳號 `defaultAccount` 路由
- `typingIndicator` 和 `resolveSenderNames` 可獨立停用
- `reactionNotifications`（`off | own | all`）
- 探針狀態快取（10 分鐘）
- **topic session 路由**：`thread_id` 作為 session scope fallback，topic key 首次穩定
- **topic 根回覆**：優先使用 `root_id` 作為 `replyTargetMessageId`
- **DM 配對回覆**：發送至 `chat:<chat_id>` 而非 `user:<open_id>`
- **Lark 私聊路由**：`chat_type: "private"` 視為 DM 上下文
- **訊息 debounce**：快速同聊天連發 debounce 成一次有序分發
- **訊息序列化**：每聊天序列化訊息處理，保持跨聊天並行
- **dedup 重啟持久化**：monitor 啟動時熱載入 dedup 狀態
- **stale 查詢抑制**：不顯示 `contact:contact.base:readonly` 非必要提示
- **非 ASCII 檔名編碼**：Feishu 多部分上傳中正確 percent-encode 中文/特殊字元檔名

## 最佳實踐

1. **錯誤處理**: 妥善處理網路錯誤和 API 限制
2. **重連機制**: 實作自動重連和指數退避
3. **速率限制**: 遵守平台 API 限制
4. **日誌記錄**: 記錄關鍵操作便於除錯
5. **安全性**: 妥善保管認證憑證
6. **串流**: 只向最終回覆傳送到外部頻道，不傳送串流/部分回覆
7. **TTS 語音氣泡**: 支援 Feishu/WhatsApp/Telegram 的語音氣泡播放（opus 輸出）
