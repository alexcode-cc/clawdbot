# 通訊頻道

## 概述

OpenClaw 支援超過 30 種訊息平台，讓使用者能透過熟悉的聊天介面與 AI 互動。頻道分為內建頻道和擴充頻道兩類。

## 內建頻道

### WhatsApp (Baileys)

**技術**: Baileys（WhatsApp Web 協議）

**特色**:
- 掃描 QR Code 登入
- 支援私訊和群組
- 媒體訊息（圖片、音訊、文件）
- 即時狀態（輸入中、已讀）

**配置**:
```json5
{
  "channels": {
    "whatsapp": {
      "enabled": true,
      "allowlist": ["+1234567890"]
    }
  }
}
```

**程式碼位置**: `src/web/`（含 `inbound/`, `auto-reply/` 子目錄）

### Telegram

**技術**: grammY（Telegram Bot API）

**特色**:
- Bot API（需要 Bot Token）
- 支援私訊和群組
- 豐富的訊息格式（Markdown、按鈕）
- Webhook 或 Polling 模式

**配置**:
```json5
{
  "channels": {
    "telegram": {
      "enabled": true,
      "token": "BOT_TOKEN",
      "allowlist": ["@username", "123456789"]
    }
  }
}
```

**程式碼位置**: `src/telegram/`

### Discord

**技術**: Carbon

**特色**:
- Bot API（需要 Bot Token）
- 支援 DM 和伺服器頻道
- 斜線命令
- 訊息附件
- 串流預覽配置（`channels.discord.streaming`）

**配置**:
```json5
{
  "channels": {
    "discord": {
      "enabled": true,
      "token": "BOT_TOKEN",
      "allowlist": ["user_id", "server_id"]
    }
  }
}
```

**程式碼位置**: `src/discord/`

### Slack

**技術**: @slack/bolt

**特色**:
- Slack App（需要 OAuth Token）
- 支援 DM 和頻道
- 互動式元件
- Webhook 模式

**配置**:
```json5
{
  "channels": {
    "slack": {
      "enabled": true,
      "token": "xoxb-...",
      "signingSecret": "...",
      "appToken": "xapp-..."
    }
  }
}
```

**程式碼位置**: `src/slack/`（含 `http/` 子目錄）

### Signal

**技術**: Signal 協議

**特色**:
- 端對端加密
- 私訊支援
- 需要 Signal CLI 或 signal-cli-rest-api

**配置**:
```json5
{
  "channels": {
    "signal": {
      "enabled": true,
      "number": "+1234567890"
    }
  }
}
```

**程式碼位置**: `src/signal/`

### iMessage

**技術**: imsg CLI（macOS 限定）

**特色**:
- macOS 原生整合
- 私訊和群組
- 需要 Full Disk Access

**配置**:
```json5
{
  "channels": {
    "imessage": {
      "enabled": true,
      "allowlist": ["+1234567890", "user@icloud.com"]
    }
  }
}
```

**程式碼位置**: `src/imessage/`

### LINE

**技術**: LINE Messaging API

**程式碼位置**: `src/line/`

## 頻道通用介面

### Channel 插件結構

```typescript
export const channelPlugin: ChannelPlugin = {
  id: 'my-channel',
  name: 'My Channel',
  
  async initialize(config: ChannelConfig) { /* 建立連接 */ },
  async startMonitor(handler: MessageHandler) { /* 監聽訊息 */ },
  async sendMessage(chatId: string, message: OutgoingMessage) { /* 發送 */ },
  async getStatus(): Promise<ChannelStatus> { return { connected: true }; },
  async shutdown() { /* 清理資源 */ },
};
```

### 訊息格式

```typescript
interface IncomingMessage {
  id: string;
  channelId: string;
  chatId: string;
  senderId: string;
  senderName?: string;
  content: string;
  attachments?: Attachment[];
  replyTo?: string;
  timestamp: Date;
}

interface OutgoingMessage {
  content: string;
  format?: 'text' | 'markdown' | 'html';
  attachments?: Attachment[];
  replyTo?: string;
  buttons?: Button[];
}
```

## 訊息路由

### Allowlist 機制

```json5
{
  "channels": {
    "telegram": {
      "allowlist": ["@user1", "@user2", "group_id"]
    }
  }
}
```

Discord allowlist 名稱解析已規範化，確保一致的比對行為。

### 串流預覽配置

統一的串流配置路徑：
```json5
{
  channels: {
    "<channel>": {
      streaming: { /* 串流預覽設定 */ }
    }
  }
}
```

### DM 範圍

CLI 本地 onboarding 預設 `session.dmScope` 為 `per-channel-peer`。

## 頻道狀態

```bash
# 基本狀態
openclaw channels status

# 全部狀態（只讀/可貼上）
openclaw channels status --all

# 深度探測
openclaw channels status --deep

# 特定頻道
openclaw channels status telegram
```

## 頻道操作

```bash
# 連接 Telegram
openclaw channels connect telegram

# 連接 WhatsApp（會顯示 QR Code）
openclaw channels connect whatsapp
```

## 開發新頻道

### 1. 建立擴充套件目錄

```
extensions/my-channel/
├── openclaw.plugin.json
├── index.ts
├── package.json
└── src/
    ├── channel.ts
    ├── monitor.ts
    ├── send.ts
    └── runtime.ts
```

### 2. 定義插件元資料

```json
// openclaw.plugin.json
{
  "id": "my-channel",
  "channels": ["my-channel"],
  "configSchema": {
    "type": "object",
    "properties": {
      "apiKey": { "type": "string" },
      "allowlist": { "type": "array", "items": { "type": "string" } }
    },
    "required": ["apiKey"]
  }
}
```

### 3. 實作頻道插件

```typescript
// src/channel.ts
import type { ChannelPlugin, MessageHandler } from 'openclaw/plugin-sdk';

export const channelPlugin: ChannelPlugin = {
  id: 'my-channel',
  name: 'My Channel',
  
  async initialize(config) { /* 初始化 SDK */ },
  async startMonitor(handler: MessageHandler) {
    client.on('message', (msg) => {
      handler({
        id: msg.id,
        channelId: 'my-channel',
        chatId: msg.chatId,
        senderId: msg.sender,
        content: msg.text,
        timestamp: new Date(),
      });
    });
  },
  async sendMessage(chatId, message) {
    await client.send(chatId, message.content);
  },
  async getStatus() { return { connected: client.isConnected() }; },
};
```

### 4. 註冊插件

```typescript
// index.ts
import type { OpenClawPluginApi } from 'openclaw/plugin-sdk';
import { channelPlugin } from './src/channel.js';

const plugin = {
  id: 'my-channel',
  name: 'My Channel',
  register(api: OpenClawPluginApi) {
    api.registerChannel({ plugin: channelPlugin });
  },
};

export default plugin;
```

### 5. 更新標籤

新增頻道/擴充套件時，需更新 `.github/labeler.yml` 並建立對應的 GitHub 標籤。

## 最佳實踐

1. **錯誤處理**: 妥善處理網路錯誤和 API 限制
2. **重連機制**: 實作自動重連和指數退避
3. **速率限制**: 遵守平台 API 限制
4. **日誌記錄**: 記錄關鍵操作便於除錯
5. **安全性**: 妥善保管認證憑證
6. **串流**: 只向最終回覆傳送到外部頻道，不傳送串流/部分回覆
