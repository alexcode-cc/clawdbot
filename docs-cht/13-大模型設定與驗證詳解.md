## 1. 模型系統概觀

OpenClaw 的模型系統支援 **26+ 個提供者前綴** 和 **900+ 個模型**，透過以下層級進行配置：

```
                    ┌─────────────────────────────────────┐
                    │         模型設定層級                  │
                    │                                     │
                    │  1. 全域預設  agents.defaults.model   │
                    │  2. Agent 覆寫  agents.list[].model  │
                    │  3. Session 覆寫  /models 指令        │
                    │  4. Fast Mode  /fast 切換            │
                    └─────────────────────────────────────┘
```

### 核心概念

| 概念 | 說明 |
|------|------|
| **Primary Model** | 主要使用的模型 |
| **Fallback Models** | 主要模型失敗時的備援清單 |
| **Auth Profile** | 認證設定檔（API Key / OAuth / Token） |
| **SecretRef** | 安全的密鑰參考機制（環境變數引用） |
| **Provider Plugin** | 提供者插件架構（Ollama/vLLM/SGLang） |
| **Fast Mode** | 優先處理推理（Anthropic `service_tier` / OpenAI `reasoning.effort`） |
| **Thinking** | 推理深度控制（`off`/`low`/`adaptive`/`high` 等） |

### API 傳輸協議

| 協議 | 使用的提供者 |
|------|------------|
| `anthropic-messages` | Anthropic、Kimi Coding |
| `openai-responses` | OpenAI、Azure OpenAI |
| `openai-codex-responses` | OpenAI Codex（OAuth） |
| `openai-completions` | 大多數 OpenAI 相容提供者 |
| `google-generative-ai` | Google Gemini |
| `bedrock-converse-stream` | Amazon Bedrock |
| `github-copilot` | GitHub Copilot |
| `ollama` | Ollama（原生 API） |

## 3. 認證設定方式

### 3.1 互動式引導（推薦新手）

```bash
# 完整互動式引導
openclaw onboard

# 僅設定模型認證
openclaw configure --section model
```

### 3.2 非互動式 CLI

```bash
# Anthropic API Key
openclaw onboard --non-interactive --accept-risk \
  --auth-choice apiKey \
  --anthropic-api-key "sk-ant-..."

# OpenAI API Key
openclaw onboard --non-interactive --accept-risk \
  --auth-choice openai-api-key \
  --openai-api-key "sk-..."

# Gemini API Key
openclaw onboard --non-interactive --accept-risk \
  --auth-choice gemini-api-key \
  --gemini-api-key "AI..."

# Ollama（本地，無需 API Key）
openclaw onboard --non-interactive --accept-risk \
  --auth-choice ollama
```

### 3.3 直接配置檔設定

編輯 `~/.openclaw/openclaw.json`：

```json5
{
  // 提供者認證
  models: {
    providers: {
      anthropic: {
        apiKey: "sk-ant-...",       // 明文（不推薦）
        // 或使用環境變數引用（推薦）
        apiKey: "${ANTHROPIC_API_KEY}",
      },
      openai: {
        apiKey: "${OPENAI_API_KEY}",
      },
      google: {
        apiKey: "${GEMINI_API_KEY}",
      },
    },
  },
}
```

### 3.4 Auth Profile 系統

OpenClaw 使用 Auth Profile 管理多個認證來源，儲存在 `~/.openclaw/agents/<agentId>/agent/auth-profiles.json`。

**認證模式**：

| 模式 | 說明 |
|------|------|
| `api_key` | 直接 API Key |
| `oauth` | OAuth 2.0 認證（Codex、Copilot、Qwen 等） |
| `token` | 供應商 CLI Token（如 Gemini CLI） |

**輪換策略**：OAuth → API Key 順序，最舊的最先使用。Session 黏著性：在 session 期間固定使用同一 profile，直到 reset/compaction/cooldown。

```bash
# 列出 auth profiles
openclaw models auth

# 互動式新增
openclaw models auth add

# OAuth 登入
openclaw models auth login

# GitHub Copilot
openclaw models auth login-github-copilot

# 貼上 Token
openclaw models auth paste-token

# 管理認證順序
openclaw models auth order get
openclaw models auth order set --provider anthropic --order "profile1,profile2"
```

### 3.5 API Key 優先順序

```
OPENCLAW_LIVE_<PROVIDER>_KEY     ← 最高優先（單一覆寫）
↓
<PROVIDER>_API_KEYS              ← 逗號/分號分隔清單
↓
<PROVIDER>_API_KEY               ← 主要環境變數
↓
<PROVIDER>_API_KEY_*             ← 編號輪換（_1, _2, ...）
↓
配置檔 models.providers.<id>.apiKey
```

## 4. 模型設定詳解

### 4.1 設定預設模型

```bash
# CLI 設定（推薦）
openclaw models set anthropic/claude-sonnet-4-5

# 查看目前設定
openclaw models status
```

### 4.2 配置檔設定

```json5
{
  agents: {
    defaults: {
      // 簡單格式
      model: "anthropic/claude-sonnet-4-5",

      // 完整格式（含 fallback）
      model: {
        primary: "anthropic/claude-sonnet-4-5",
        fallbacks: [
          "anthropic/claude-opus-4-5",
          "openai/gpt-5.4",
          "ollama/qwen2.5-coder:32b",
        ],
      },

      // 圖像模型
      imageModel: "openai/gpt-image-1",

      // PDF 分析模型
      pdfModel: "anthropic/claude-sonnet-4-5",
    },
  },
}
```

### 4.3 模型別名

```bash
# 新增別名
openclaw models aliases add --alias "opus" --model "anthropic/claude-opus-4-6"
openclaw models aliases add --alias "sonnet" --model "anthropic/claude-sonnet-4-5"

# 列出別名
openclaw models aliases list

# 移除別名
openclaw models aliases remove --alias "opus"
```

配置檔：

```json5
{
  agents: {
    defaults: {
      models: {
        "anthropic/claude-opus-4-6": { alias: "Opus" },
        "anthropic/claude-sonnet-4-5": { alias: "Sonnet" },
        "ollama/qwen2.5-coder:32b": { alias: "Qwen 32B" },
      },
    },
  },
}
```

### 4.4 Per-Agent 模型覆寫

```json5
{
  agents: {
    list: [
      {
        id: "coding",
        model: "anthropic/claude-opus-4-6",
      },
      {
        id: "chat",
        model: "ollama/llama3.3:70b",
        params: {
          thinking: "off",
          temperature: 0.7,
        },
      },
    ],
  },
}
```

### 4.5 列出可用模型

```bash
# 已配置的模型
openclaw models list

# 所有可用模型（794+）
openclaw models list --all

# 按提供者篩選
openclaw models list --provider anthropic
openclaw models list --provider openai
openclaw models list --provider ollama

# 僅本地模型
openclaw models list --local

# JSON 格式
openclaw models list --json

# 純文字格式
openclaw models list --plain
```

## 5. Anthropic (Claude) 設定

### 認證方式

| 方式 | 引導選項 | 說明 |
|------|---------|------|
| API Key | `--auth-choice apiKey` | 直接 API Key |
| OAuth Token | `--auth-choice token` | Anthropic OAuth |
| 環境變數 | — | `ANTHROPIC_API_KEY` |

### 設定步驟

```bash
# 方式一：互動式
openclaw onboard
# 選擇 Anthropic → API Key → 輸入 Key

# 方式二：非互動式
openclaw onboard --non-interactive --accept-risk \
  --auth-choice apiKey \
  --anthropic-api-key "sk-ant-api03-..."

# 方式三：環境變數
export ANTHROPIC_API_KEY="sk-ant-api03-..."
openclaw onboard --non-interactive --accept-risk \
  --auth-choice apiKey \
  --secret-input-mode ref

# 方式四：直接設定模型
openclaw models set anthropic/claude-sonnet-4-5
```

### 可用模型

| 模型 | Context | 特色 |
|------|---------|------|
| `anthropic/claude-opus-4-6` | 200k | 最強推理 |
| `anthropic/claude-sonnet-4-6` | 200k | 平衡效能 |
| `anthropic/claude-sonnet-4-5` | 200k | 上一代 |
| `anthropic/claude-haiku-4-5` | 200k | 最快速 |

### Thinking 模式

Claude 4.6 預設使用 `adaptive` 思考模式：

```json5
{
  agents: {
    defaults: {
      thinkingDefault: "adaptive", // adaptive | low | off
    },
  },
}
```

### Fast Mode

```json5
{
  agents: {
    defaults: {
      models: {
        "anthropic/claude-opus-4-6": {
          params: { fastMode: true },
        },
      },
    },
  },
}
```

啟用後：`service_tier: "auto"`（優先佇列）。僅適用於直接 API Key。

```bash
# 查看模型狀態
openclaw models status

# 探測認證
openclaw models status --probe

# 檢查過期
openclaw models status --check

# 快速測試
openclaw agent --message "Hello" --json
```

## 7. Google Gemini 設定

### 認證方式一覽

Google 提供多種存取 Gemini 模型的方式，但它們是**不同的產品**，適用的認證方式也不同：

| 認證方式 | 引導選項 | 適用對象 | 需要 GCP 專案 |
|---------|---------|---------|:----------:|
| **Gemini API Key**（推薦） | `--auth-choice gemini-api-key` | 所有使用者（含 Gemini Pro 訂閱用戶） | 否 |
| Gemini CLI OAuth | `--auth-choice google-gemini-cli` | Google Cloud Code Assist 企業訂閱用戶 | **是** |
| Vertex AI | 手動配置 | Google Cloud 服務帳號 | **是** |

> **重要區分**：Google Gemini Pro（Google One AI Premium）訂閱 與 Google Cloud Code Assist 是**完全不同的產品**。Gemini Pro 訂閱用戶應使用 **Gemini API Key** 方式，而非 Gemini CLI OAuth。

| | Google Gemini Pro 訂閱 | Google Cloud Code Assist |
|---|---|---|
| **產品定位** | 消費者 AI 助手（Google One AI Premium） | 企業開發者工具（GCP） |
| **使用場景** | gemini.google.com、Gmail、Docs、API | IDE/CLI 程式碼助手 |
| **API 端點** | `generativelanguage.googleapis.com` | `cloudcode-pa.googleapis.com` |
| **認證方式** | API Key（AI Studio 取得） | GCP 專案 + Code Assist 訂閱 |
| **OpenClaw 選項** | `--auth-choice gemini-api-key` | `--auth-choice google-gemini-cli` |

---

### Gemini Pro 訂閱用戶：使用 API Key 設定（推薦）

Gemini Pro（Google One AI Premium）訂閱用戶透過 **API Key** 存取 Gemini API 時，享有比免費用戶**更高的 API 配額**（RPM/TPM），且支援所有 Gemini 模型。以下是完整設定步驟：

#### 步驟 1：取得 Gemini API Key

1. 前往 **Google AI Studio**：<https://aistudio.google.com/apikey>
2. 使用你的 **Gemini Pro 訂閱帳號**（Google One AI Premium 帳號）登入
3. 點擊 **「Create API Key」**（建立 API 金鑰）
4. 選擇一個現有的 Google Cloud 專案，或讓系統自動建立一個（這裡的 GCP 專案僅用於 API Key 管理，不需要額外的付費訂閱）
5. 複製產生的 API Key（格式如 `AIza...`）

> **提示**：API Key 頁面也會顯示你目前的配額狀態。Gemini Pro 訂閱用戶的配額顯著高於免費層級。

#### 步驟 2：在 OpenClaw 中設定

**方式 A：互動式設定（推薦新手）**

```bash
openclaw configure --section model
# 選擇 Google → Gemini API Key → 貼上你的 API Key
```

**方式 B：引導精靈設定**

```bash
openclaw onboard
# 在認證選擇中選擇「Google」→「Gemini API Key」
# 貼上你的 API Key
```

**方式 C：非互動式設定（推薦自動化）**

```bash
# 直接使用 API Key
openclaw onboard --non-interactive --accept-risk \
  --auth-choice gemini-api-key \
  --gemini-api-key "AIza..."

# 或使用環境變數（更安全，推薦）
export GEMINI_API_KEY="AIza..."
openclaw onboard --non-interactive --accept-risk \
  --auth-choice gemini-api-key \
  --secret-input-mode ref
```

**方式 D：直接編輯配置檔**

```json5
// ~/.openclaw/openclaw.json
{
  models: {
    providers: {
      google: {
        apiKey: "${GEMINI_API_KEY}",  // 環境變數引用（推薦）
        // 或直接明文（不推薦用於生產環境）
        // apiKey: "AIza...",
      },
    },
  },
}
```

#### 步驟 3：設定預設模型

```bash
# 推薦：使用 Flash 模型（Free tier 配額較高）
openclaw models set google/gemini-2.5-flash

# 或使用 Pro 模型（需要 Pay-as-you-go 計費，見下方配額說明）
openclaw models set google/gemini-2.5-pro
```

> **重要**：Free tier 下 `gemini-2.5-pro` 的配額極低（RPM/TPM limit 可能為 0），建議先用 `gemini-2.5-flash` 測試成功後再考慮升級。

#### 步驟 4：驗證設定

```bash
# 查看模型狀態
openclaw models status
# 應顯示 google/gemini-2.5-flash 為預設模型

# 探測認證是否有效
openclaw models status --probe
# 應顯示 google 提供者認證正常

# 快速測試對話
openclaw tui
# 發送 hello 測試
# 成功時應在數秒內收到回覆
```

#### 步驟 5：設定 Fallback（可選但推薦）

```bash
# 新增 Pro 作為 fallback（如已啟用付費計費）
openclaw models fallbacks add google/gemini-2.5-pro
```

#### 重要：API 配額與計費層級

> **Gemini Pro 月費訂閱（Google One AI Premium）不包含 API 呼叫額度。** API 呼叫使用獨立的計費系統。

#### 常見問題

**Q: 我已經有 Gemini Pro 訂閱，為什麼 Gemini CLI OAuth 失敗？**

A: Gemini CLI OAuth 使用的是 Google Cloud Code Assist API（`cloudcode-pa.googleapis.com`），與 Gemini API Key 走的路徑（`generativelanguage.googleapis.com`）完全不同。即使你的帳號可以登入 Google Antigravity 進行 AI 編程，Gemini CLI OAuth 仍可能失敗（見下方 Gemini CLI OAuth 排錯指南）。**建議優先使用 API Key 方式**，功能和模型完全相同且更穩定。

**Q: 我有 Gemini Pro 月費，API 呼叫會用到我的月費額度嗎？**

A: **不會**。Gemini Pro 月費（Google One AI Premium）和 Gemini API Key 是**完全獨立的計費系統**。API Key 預設是 Free tier，有自己的免費額度。要提高 API 配額需要在 GCP 專案上另外連結帳單帳戶（Pay-as-you-go），費用按 token 用量計費，與月費訂閱無關。

**Q: Free tier 的 `gemini-2.5-pro` 配額為 0 怎麼辦？**

A: 這是 Google 的 Free tier 限制。解決方案：
1. **改用 `gemini-2.5-flash`**（Free tier 有免費配額，效能也很好）
2. **啟用 Pay-as-you-go**：在 GCP Console 連結帳單帳戶後，`gemini-2.5-pro` 配額會大幅提高

**Q: 為什麼在 TUI 中切換模型時出現 `model not allowed`？**

A: 模型需要在 configured models allowlist 中才能在 TUI 中切換。解決方法：
```bash
# 退出 TUI 後執行
openclaw models set google/gemini-2.5-flash
openclaw gateway restart
```
或重新執行 `openclaw configure --section model` 在模型選擇器中勾選需要的模型。

**Q: 環境變數 `GEMINI_API_KEY` 和配置檔 `apiKey` 哪個優先？**

A: 環境變數優先。優先順序為：`OPENCLAW_LIVE_GOOGLE_KEY` > `GEMINI_API_KEY` > 配置檔 `models.providers.google.apiKey`。

**Q: `google/gemini-2.5-pro` 和 `google-gemini-cli/gemini-2.5-pro` 有什麼差別？**

A: 是**同一個底層模型**，差別只在認證路徑：

| | `google/` 前綴 | `google-gemini-cli/` 前綴 |
|---|---|---|
| API 端點 | `generativelanguage.googleapis.com` | `cloudcode-pa.googleapis.com` |
| 認證方式 | API Key | Code Assist OAuth |
| 消費者帳號 | **完全相容** | 不穩定（可能 400） |
| 推薦程度 | **推薦** | 僅限企業 Code Assist 用戶 |

#### 排錯：常見錯誤與解決方案

##### 錯誤 1：`Gemini CLI not found`

```
Error: Gemini CLI not found. Install it first: brew install gemini-cli
```

**解決方案**：Checklist 步驟 1-2，安裝 Gemini CLI。或手動指定 OAuth Client：

```bash
export GEMINI_CLI_OAUTH_CLIENT_ID="your-client-id"
```

##### 錯誤 2：`loadCodeAssist failed: 400 Bad Request`

```
Error: loadCodeAssist failed: 400 Bad Request
    at discoverProject (extensions/google-gemini-cli-auth/oauth.ts:514:21)
```

**發生時機**：OAuth token exchange 成功，但 `discoverProject()` 呼叫 Code Assist API 時失敗。

**解決方案**：Checklist 步驟 8-11，設定 `GOOGLE_CLOUD_PROJECT` 環境變數：

```bash
export GOOGLE_CLOUD_PROJECT="your-project-id"
openclaw models auth login --provider google-gemini-cli --set-default
```

##### 錯誤 3：`Gemini for Google Cloud API has not been used in project ... or it is disabled` (403)

**解決方案**：Checklist 步驟 9，在 GCP Console 啟用 **Cloud AI Companion API**（又稱 Gemini for Google Cloud）：

1. 前往 <https://console.cloud.google.com/apis/library/cloudaicompanion.googleapis.com>
2. 點擊 **「啟用」(Enable)**
3. 等待約 1 分鐘後重試

> **注意**：啟用 API 可能需要 GCP 專案已連結帳單帳戶。

##### 錯誤 4：`Cloud Code Assist API error (400): Request contains an invalid argument`

```
run error: Cloud Code Assist API error (400): Request contains an invalid argument.
```

**發生時機**：OAuth 認證和專案設定都成功了（`openclaw models status` 顯示正常、token 未過期），但實際呼叫 AI 模型時失敗。

**實際排錯經驗紀錄**（2026-03-15 驗證）：

以下是經過實際測試的完整排查流程和結果：

| 排查步驟 | 操作 | 結果 |
|---------|------|------|
| 1. 切換穩定版模型 | `openclaw models set google-gemini-cli/gemini-2.5-pro` | 仍然 400 |
| 2. TUI 中關閉 thinking | `/thinking off` | 仍然 400 |
| 3. 全域永久關閉 thinking | `openclaw config set agents.defaults.thinkingDefault "off"` + gateway restart | 仍然 400（等待 10 分鐘逾時） |
| 4. 切換所有 gemini-cli 模型 | 測試 `gemini-2.5-flash`、`gemini-2.5-pro`、`gemini-3.1-pro-preview` | **全部** 400 |
| 5. 切換 Ollama Cloud 模型 | `ollama/kimi-k2.5:cloud` | **成功**（確認 Gateway 和 session 本身正常） |
| 6. 檢查 auth profile | `cat ~/.openclaw/agents/main/agent/auth-profiles.json` | 發現殘留的無效 `google-gemini-cli:default` profile |
| 7. 移除無效 profile | `jq 'del(.profiles["google-gemini-cli:default"])' ...` | doctor 警告消失 |
| 8. 查看 Gateway 日誌 | `tail -100 /tmp/openclaw/openclaw-*.log` | 發現 fallback 到 `gemini-2.5-flash` 時其實**曾經成功過** |

**根因分析**：

Code Assist API（`cloudcode-pa.googleapis.com`）對 **Gemini Pro 消費者帳號**（Google One AI Premium）的請求格式存在不相容問題。雖然 OAuth 認證能成功完成，API 也接受 token，但實際的模型推理呼叫會回傳 `400: invalid argument`。這不是 OpenClaw 的 bug，而是 **Code Assist API 主要設計給企業 Google Cloud Code Assist 訂閱用戶**，消費者帳號的支援有限且不穩定。

**關鍵發現**：
- OAuth 認證和專案佈建（`loadCodeAssist`、`onboardUser`）可以成功
- `openclaw models status` 顯示 token 有效、用量 100% 可用
- 但所有 `google-gemini-cli/*` 模型的推理呼叫都回傳 400
- 同一 Gateway 上的其他提供者（Ollama Cloud）完全正常
- Gateway 日誌中 fallback 機制偶爾會讓 `gemini-2.5-flash` 成功，但不穩定

**結論**：Gemini Pro 消費者帳號應使用 **Gemini API Key 方式**（`google/gemini-2.5-pro`），不應使用 Gemini CLI OAuth 方式（`google-gemini-cli/gemini-2.5-pro`）。兩者存取相同的底層模型，但 API Key 方式走 `generativelanguage.googleapis.com`，對消費者帳號完全相容。

**解決方案**：

```bash
# 改用 Gemini API Key（推薦）
# 1. 從 https://aistudio.google.com/apikey 取得 API Key
export GEMINI_API_KEY="AIza..."
echo 'export GEMINI_API_KEY="AIza..."' >> ~/.zshrc

# 2. 設定
openclaw configure --section model
# 選擇 Google → Gemini API Key

# 3. 切換模型路徑
openclaw models set google/gemini-2.5-pro

# 4. 恢復 thinking（API Key 方式支援）
openclaw config set agents.defaults.thinkingDefault "adaptive"

# 5. 重啟並測試
openclaw gateway restart
openclaw tui
```

```
doctor: google-gemini-cli:default: missing — Re-auth via openclaw configure
```

**解決方案**：

```bash
# 重啟並確認
openclaw gateway restart
openclaw doctor
# 確認不再顯示 "google-gemini-cli:default: missing"
```

### 觸發 Fallback 的條件

| 條件 | 說明 |
|------|------|
| 網路錯誤 | `ECONNREFUSED`、`ENETUNREACH`、`EHOSTUNREACH` 等 |
| HTTP 402 | Billing 錯誤（Venice、Poe、OpenRouter credits） |
| HTTP 429 | Rate limit |
| HTTP 503 | Service unavailable（需含 overload indicator） |
| 認證過期 | OAuth/Token 過期後自動刷新並重試 |
| Thinking 不支援 | 自動降級為 `think=off` |

## 15. SecretRef 安全儲存

### 明文 vs 參考模式

| 模式 | 說明 | 適用場景 |
|------|------|---------|
| `plaintext` | API Key 直接存入配置檔 | 個人開發 |
| `ref` | 存為環境變數參考 | 生產環境、CI/CD |

### 使用 SecretRef

```bash
# 引導時使用 ref 模式
openclaw onboard --non-interactive --accept-risk \
  --auth-choice apiKey \
  --anthropic-api-key "$ANTHROPIC_API_KEY" \
  --secret-input-mode ref
```

配置檔中的 SecretRef：

```json5
{
  models: {
    providers: {
      anthropic: {
        apiKey: "${ANTHROPIC_API_KEY}", // 環境變數引用
      },
    },
  },
  gateway: {
    auth: {
      token: { "$ref": "secret:gateway-token" }, // SecretRef 引用
    },
  },
}
```

### 管理 Secrets

```bash
# 重新載入 secrets
openclaw secrets reload

# 審計明文密鑰
openclaw secrets audit

# 互動式設定
openclaw secrets configure
```

|------|------|---------|
