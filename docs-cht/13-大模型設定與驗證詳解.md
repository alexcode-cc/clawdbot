## 1. 模型系統概觀

OpenClaw 的模型系統支援 **23 個提供者前綴** 和 **794+ 個模型**，透過以下層級進行配置：

```
                    ┌─────────────────────────────────────┐
                    │         模型設定層級                  │
                    │                                     │
                    │  1. 全域預設  agents.defaults.model   │
                    │  2. Agent 覆寫  agents.list[].model  │
                    │  3. Session 覆寫  /models 指令        │
                    │  4. Fast Mode  /fast 切換            │
                    └─────────────────────────────────────┘
```

### 核心概念

| 概念 | 說明 |
|------|------|
| **Primary Model** | 主要使用的模型 |
| **Fallback Models** | 主要模型失敗時的備援清單 |
| **Auth Profile** | 認證設定檔（API Key / OAuth / Token） |
| **SecretRef** | 安全的密鑰參考機制（環境變數引用） |
| **Provider Plugin** | 提供者插件架構（Ollama/vLLM/SGLang） |
| **Fast Mode** | 優先處理推理（Anthropic `service_tier` / OpenAI `reasoning.effort`） |
| **Thinking** | 推理深度控制（`off`/`low`/`adaptive`/`high` 等） |

### API 傳輸協議

| 協議 | 使用的提供者 |
|------|------------|
| `anthropic-messages` | Anthropic、Kimi Coding |
| `openai-responses` | OpenAI、Azure OpenAI |
| `openai-codex-responses` | OpenAI Codex（OAuth） |
| `openai-completions` | 大多數 OpenAI 相容提供者 |
| `google-generative-ai` | Google Gemini |
| `bedrock-converse-stream` | Amazon Bedrock |
| `github-copilot` | GitHub Copilot |
| `ollama` | Ollama（原生 API） |

## 3. 認證設定方式

### 3.1 互動式引導（推薦新手）

```bash
# 完整互動式引導
openclaw onboard

# 僅設定模型認證
openclaw configure --section model
```

### 3.2 非互動式 CLI

```bash
# Anthropic API Key
openclaw onboard --non-interactive --accept-risk \
  --auth-choice apiKey \
  --anthropic-api-key "sk-ant-..."

# OpenAI API Key
openclaw onboard --non-interactive --accept-risk \
  --auth-choice openai-api-key \
  --openai-api-key "sk-..."

# Gemini API Key
openclaw onboard --non-interactive --accept-risk \
  --auth-choice gemini-api-key \
  --gemini-api-key "AI..."

# Ollama（本地，無需 API Key）
openclaw onboard --non-interactive --accept-risk \
  --auth-choice ollama
```

### 3.3 直接配置檔設定

編輯 `~/.openclaw/openclaw.json`：

```json5
{
  // 提供者認證
  models: {
    providers: {
      anthropic: {
        apiKey: "sk-ant-...",       // 明文（不推薦）
        // 或使用環境變數引用（推薦）
        apiKey: "${ANTHROPIC_API_KEY}",
      },
      openai: {
        apiKey: "${OPENAI_API_KEY}",
      },
      google: {
        apiKey: "${GEMINI_API_KEY}",
      },
    },
  },
}
```

### 3.4 Auth Profile 系統

OpenClaw 使用 Auth Profile 管理多個認證來源，儲存在 `~/.openclaw/agents/<agentId>/agent/auth-profiles.json`。

**認證模式**：

| 模式 | 說明 |
|------|------|
| `api_key` | 直接 API Key |
| `oauth` | OAuth 2.0 認證（Codex、Copilot、Qwen 等） |
| `token` | 供應商 CLI Token（如 Gemini CLI） |

**輪換策略**：OAuth → API Key 順序，最舊的最先使用。Session 黏著性：在 session 期間固定使用同一 profile，直到 reset/compaction/cooldown。

```bash
# 列出 auth profiles
openclaw models auth

# 互動式新增
openclaw models auth add

# OAuth 登入
openclaw models auth login

# GitHub Copilot
openclaw models auth login-github-copilot

# 貼上 Token
openclaw models auth paste-token

# 管理認證順序
openclaw models auth order get
openclaw models auth order set --provider anthropic --order "profile1,profile2"
```

### 3.5 API Key 優先順序

```
OPENCLAW_LIVE_<PROVIDER>_KEY     ← 最高優先（單一覆寫）
↓
<PROVIDER>_API_KEYS              ← 逗號/分號分隔清單
↓
<PROVIDER>_API_KEY               ← 主要環境變數
↓
<PROVIDER>_API_KEY_*             ← 編號輪換（_1, _2, ...）
↓
配置檔 models.providers.<id>.apiKey
```

## 4. 模型設定詳解

### 4.1 設定預設模型

```bash
# CLI 設定（推薦）
openclaw models set anthropic/claude-sonnet-4-5

# 查看目前設定
openclaw models status
```

### 4.2 配置檔設定

```json5
{
  agents: {
    defaults: {
      // 簡單格式
      model: "anthropic/claude-sonnet-4-5",

      // 完整格式（含 fallback）
      model: {
        primary: "anthropic/claude-sonnet-4-5",
        fallbacks: [
          "anthropic/claude-opus-4-5",
          "openai/gpt-5.4",
          "ollama/qwen2.5-coder:32b",
        ],
      },

      // 圖像模型
      imageModel: "openai/gpt-image-1",

      // PDF 分析模型
      pdfModel: "anthropic/claude-sonnet-4-5",
    },
  },
}
```

### 4.3 模型別名

```bash
# 新增別名
openclaw models aliases add --alias "opus" --model "anthropic/claude-opus-4-6"
openclaw models aliases add --alias "sonnet" --model "anthropic/claude-sonnet-4-5"

# 列出別名
openclaw models aliases list

# 移除別名
openclaw models aliases remove --alias "opus"
```

配置檔：

```json5
{
  agents: {
    defaults: {
      models: {
        "anthropic/claude-opus-4-6": { alias: "Opus" },
        "anthropic/claude-sonnet-4-5": { alias: "Sonnet" },
        "ollama/qwen2.5-coder:32b": { alias: "Qwen 32B" },
      },
    },
  },
}
```

### 4.4 Per-Agent 模型覆寫

```json5
{
  agents: {
    list: [
      {
        id: "coding",
        model: "anthropic/claude-opus-4-6",
      },
      {
        id: "chat",
        model: "ollama/llama3.3:70b",
        params: {
          thinking: "off",
          temperature: 0.7,
        },
      },
    ],
  },
}
```

### 4.5 列出可用模型

```bash
# 已配置的模型
openclaw models list

# 所有可用模型（794+）
openclaw models list --all

# 按提供者篩選
openclaw models list --provider anthropic
openclaw models list --provider openai
openclaw models list --provider ollama

# 僅本地模型
openclaw models list --local

# JSON 格式
openclaw models list --json

# 純文字格式
openclaw models list --plain
```

## 5. Anthropic (Claude) 設定

### 認證方式

| 方式 | 引導選項 | 說明 |
|------|---------|------|
| API Key | `--auth-choice apiKey` | 直接 API Key |
| OAuth Token | `--auth-choice token` | Anthropic OAuth |
| 環境變數 | — | `ANTHROPIC_API_KEY` |

### 設定步驟

```bash
# 方式一：互動式
openclaw onboard
# 選擇 Anthropic → API Key → 輸入 Key

# 方式二：非互動式
openclaw onboard --non-interactive --accept-risk \
  --auth-choice apiKey \
  --anthropic-api-key "sk-ant-api03-..."

# 方式三：環境變數
export ANTHROPIC_API_KEY="sk-ant-api03-..."
openclaw onboard --non-interactive --accept-risk \
  --auth-choice apiKey \
  --secret-input-mode ref

# 方式四：直接設定模型
openclaw models set anthropic/claude-sonnet-4-5
```

### 可用模型

| 模型 | Context | 特色 |
|------|---------|------|
| `anthropic/claude-opus-4-6` | 200k | 最強推理 |
| `anthropic/claude-sonnet-4-6` | 200k | 平衡效能 |
| `anthropic/claude-sonnet-4-5` | 200k | 上一代 |
| `anthropic/claude-haiku-4-5` | 200k | 最快速 |

### Thinking 模式

Claude 4.6 預設使用 `adaptive` 思考模式：

```json5
{
  agents: {
    defaults: {
      thinkingDefault: "adaptive", // adaptive | low | off
    },
  },
}
```

### Fast Mode

```json5
{
  agents: {
    defaults: {
      models: {
        "anthropic/claude-opus-4-6": {
          params: { fastMode: true },
        },
      },
    },
  },
}
```

啟用後：`service_tier: "auto"`（優先佇列）。僅適用於直接 API Key。

```bash

openclaw models status --probe

# 檢查過期
openclaw models status --check

# 快速測試
openclaw agent --message "Hello" --json
```

## 7. Google Gemini 設定

### 認證方式一覽

Google 提供多種存取 Gemini 模型的方式，但它們是**不同的產品**，適用的認證方式也不同：

| 認證方式 | 引導選項 | 適用對象 | 需要 GCP 專案 |
|---------|---------|---------|:----------:|
| **Gemini API Key**（推薦） | `--auth-choice gemini-api-key` | 所有使用者（含 Gemini Pro 訂閱用戶） | 否 |
| Gemini CLI OAuth | `--auth-choice google-gemini-cli` | Google Cloud Code Assist 企業訂閱用戶 | **是** |
| Vertex AI | 手動配置 | Google Cloud 服務帳號 | **是** |

> **重要區分**：Google Gemini Pro（Google One AI Premium）訂閱 與 Google Cloud Code Assist 是**完全不同的產品**。Gemini Pro 訂閱用戶應使用 **Gemini API Key** 方式，而非 Gemini CLI OAuth。

| | Google Gemini Pro 訂閱 | Google Cloud Code Assist |
|---|---|---|
| **產品定位** | 消費者 AI 助手（Google One AI Premium） | 企業開發者工具（GCP） |
| **使用場景** | gemini.google.com、Gmail、Docs、API | IDE/CLI 程式碼助手 |
| **API 端點** | `generativelanguage.googleapis.com` | `cloudcode-pa.googleapis.com` |
| **認證方式** | API Key（AI Studio 取得） | GCP 專案 + Code Assist 訂閱 |
| **OpenClaw 選項** | `--auth-choice gemini-api-key` | `--auth-choice google-gemini-cli` |

### Gemini Pro 訂閱用戶：使用 API Key 設定（推薦）

Gemini Pro（Google One AI Premium）訂閱用戶透過 **API Key** 存取 Gemini API 時，享有比免費用戶**更高的 API 配額**（RPM/TPM），且支援所有 Gemini 模型。以下是完整設定步驟：

#### 步驟 1：取得 Gemini API Key

1. 前往 **Google AI Studio**：<https://aistudio.google.com/apikey>
2. 使用你的 **Gemini Pro 訂閱帳號**（Google One AI Premium 帳號）登入
3. 點擊 **「Create API Key」**（建立 API 金鑰）
4. 選擇一個現有的 Google Cloud 專案，或讓系統自動建立一個（這裡的 GCP 專案僅用於 API Key 管理，不需要額外的付費訂閱）
5. 複製產生的 API Key（格式如 `AIza...`）

> **提示**：API Key 頁面也會顯示你目前的配額狀態。Gemini Pro 訂閱用戶的配額顯著高於免費層級。

#### 步驟 2：在 OpenClaw 中設定

**方式 A：互動式設定（推薦新手）**

```bash
openclaw configure --section model
# 選擇 Google → Gemini API Key → 貼上你的 API Key
```

**方式 B：引導精靈設定**

```bash
openclaw onboard
# 在認證選擇中選擇「Google」→「Gemini API Key」
# 貼上你的 API Key
```

**方式 C：非互動式設定（推薦自動化）**

```bash
# 直接使用 API Key
openclaw onboard --non-interactive --accept-risk \
  --auth-choice gemini-api-key \
  --gemini-api-key "AIza..."

# 或使用環境變數（更安全，推薦）
export GEMINI_API_KEY="AIza..."
openclaw onboard --non-interactive --accept-risk \
  --auth-choice gemini-api-key \
  --secret-input-mode ref
```

**方式 D：直接編輯配置檔**

```json5
// ~/.openclaw/openclaw.json
{
  models: {
    providers: {
      google: {
        apiKey: "${GEMINI_API_KEY}",  // 環境變數引用（推薦）
        // 或直接明文（不推薦用於生產環境）
        // apiKey: "AIza...",
      },
    },
  },
}
```

#### 步驟 3：設定預設模型

```bash
# 設定 Gemini 2.5 Pro 為預設模型
openclaw models set google/gemini-2.5-pro

# 或使用快速的 Flash 模型
openclaw models set google/gemini-2.5-flash
```

#### 步驟 4：驗證設定

```bash
# 查看模型狀態

# 應顯示 google/gemini-2.5-pro 為預設模型

# 探測認證是否有效
openclaw models status --probe
# 應顯示 google 提供者認證正常

# 快速測試對話

# 如果想在 TUI 中測試
```

#### 步驟 5：設定 Fallback（可選但推薦）

```bash
# 新增 Flash 作為 fallback（更快、更便宜）

```

#### 常見問題

**Q: 我已經有 Gemini Pro 訂閱，為什麼 Gemini CLI OAuth 失敗？**

A: Gemini CLI OAuth 使用的是 Google Cloud Code Assist 服務（`cloudcode-pa.googleapis.com`），這是一個**獨立的企業開發者產品**，與消費者的 Gemini Pro 訂閱（Google One AI Premium）**完全無關**。請改用 API Key 方式。

**Q: API Key 和 Gemini Pro 訂閱的配額有關嗎？**

A: 是的。API Key 繫結到你的 Google 帳號，如果該帳號有 Gemini Pro 訂閱（Google One AI Premium），API 配額會自動提高。你不需要額外設定。

**Q: 我需要為 API Key 付費嗎？**

A: API Key 本身是免費的。如果你已經有 Gemini Pro 訂閱，API 呼叫包含在你的訂閱配額內。超出免費配額的部分可能會按用量計費（取決於你的 Google Cloud 設定）。

**Q: 環境變數 `GEMINI_API_KEY` 和配置檔 `apiKey` 哪個優先？**

A: 環境變數優先。優先順序為：`OPENCLAW_LIVE_GOOGLE_KEY` > `GEMINI_API_KEY` > 配置檔 `models.providers.google.apiKey`。

## 11. 模型 Fallback 與容錯

### Fallback 設定

```bash
# 新增 fallback
openclaw models fallbacks add anthropic/claude-opus-4-5
openclaw models fallbacks add openai/gpt-5.4
openclaw models fallbacks add ollama/qwen2.5-coder:32b

# 列出
openclaw models fallbacks list

# 移除
openclaw models fallbacks remove openai/gpt-5.4

# 清除所有
openclaw models fallbacks clear
```

### 配置檔

```json5
{
  agents: {
    defaults: {
      model: {
        primary: "anthropic/claude-sonnet-4-5",
        fallbacks: [
          "anthropic/claude-opus-4-5",
          "openai/gpt-5.4",
          "ollama/qwen2.5-coder:32b",
        ],
      },
    },
  },
}
```

### 觸發 Fallback 的條件

| 條件 | 說明 |
|------|------|
| 網路錯誤 | `ECONNREFUSED`、`ENETUNREACH`、`EHOSTUNREACH` 等 |
| HTTP 402 | Billing 錯誤（Venice、Poe、OpenRouter credits） |
| HTTP 429 | Rate limit |
| HTTP 503 | Service unavailable（需含 overload indicator） |
| 認證過期 | OAuth/Token 過期後自動刷新並重試 |
| Thinking 不支援 | 自動降級為 `think=off` |

### 圖像模型 Fallback

```bash
openclaw models image-fallbacks add openai/gpt-image-1
openclaw models image-fallbacks list
```

## 15. SecretRef 安全儲存

### 明文 vs 參考模式

| 模式 | 說明 | 適用場景 |
|------|------|---------|
| `plaintext` | API Key 直接存入配置檔 | 個人開發 |
| `ref` | 存為環境變數參考 | 生產環境、CI/CD |

### 使用 SecretRef

```bash
# 引導時使用 ref 模式
openclaw onboard --non-interactive --accept-risk \
  --auth-choice apiKey \
  --anthropic-api-key "$ANTHROPIC_API_KEY" \
  --secret-input-mode ref
```

配置檔中的 SecretRef：

```json5
{
  models: {
    providers: {
      anthropic: {
        apiKey: "${ANTHROPIC_API_KEY}", // 環境變數引用
      },
    },
  },
  gateway: {
    auth: {
      token: { "$ref": "secret:gateway-token" }, // SecretRef 引用
    },
  },
}
```

### 管理 Secrets

```bash
# 重新載入 secrets
openclaw secrets reload

# 審計明文密鑰
openclaw secrets audit

# 互動式設定
openclaw secrets configure
```

### 錯誤代碼對照

| 錯誤 | 原因 | 解決方式 |
|------|------|---------|
| HTTP 401 | API Key 無效或過期 | 重新設定 `openclaw models auth add` |
| HTTP 402 | 餘額不足 | 充值或設定 fallback 模型 |
| HTTP 429 | Rate limit | 等待或切換提供者 |
| HTTP 400 (Azure) | 內容過濾器 | 已在 2026.3.13 修復 |
| `ECONNREFUSED` | 本地模型未啟動 | 啟動 Ollama/vLLM |
| Unknown model | 模型 ID 錯誤 | 使用 `openclaw models list --all` 查看可用模型 |
