# 擴充套件開發

## 概述

OpenClaw 擴充套件（插件）是獨立的模組，可新增頻道、工具、CLI 命令等功能。本文件說明如何開發和發布擴充套件。

## 擴充套件結構

### 目錄結構

```
extensions/<plugin-id>/
├── openclaw.plugin.json   # 插件元資料
├── index.ts               # 入口檔案
├── package.json           # 依賴管理
├── README.md              # 說明文件
├── CHANGELOG.md           # 變更記錄
└── src/                   # 原始碼
    ├── channel.ts         # 頻道實作（如適用）
    ├── runtime.ts         # 運行時設定
    ├── monitor.ts         # 訊息監控
    ├── send.ts            # 發送邏輯
    ├── onboarding.ts      # 引導流程
    ├── tool.ts            # 工具實作（如適用）
    └── types.ts           # 類型定義
```

### 元資料檔案

```json
// openclaw.plugin.json
{
  "id": "my-plugin",
  "name": "My Plugin",
  "version": "1.0.0",
  "description": "A custom plugin for OpenClaw",
  
  "channels": ["my-channel"],
  "providers": ["my-provider"],
  "kind": "channel",  // channel | memory | tool | provider
  
  "configSchema": {
    "type": "object",
    "properties": {
      "apiKey": {
        "type": "string",
        "description": "API Key for the service"
      },
      "enabled": {
        "type": "boolean",
        "default": true
      }
    },
    "required": ["apiKey"]
  }
}
```

### 入口檔案

```typescript
// index.ts
import type { OpenClawPluginApi, OpenClawPlugin } from 'openclaw/plugin-sdk';
import { emptyPluginConfigSchema } from 'openclaw/plugin-sdk';
import { channelPlugin } from './src/channel.js';
import { setRuntime } from './src/runtime.js';

const plugin: OpenClawPlugin = {
  id: 'my-plugin',
  name: 'My Plugin',
  description: 'A custom plugin',
  configSchema: emptyPluginConfigSchema(),
  
  register(api: OpenClawPluginApi) {
    setRuntime(api.runtime);
    api.registerChannel({ plugin: channelPlugin });
    api.registerTool(myTool);
    api.registerCli(registerMyCli);
    api.registerGatewayMethod('my-plugin.action', myHandler);
    // ⚠️ registerHttpHandler 已移除，請改用 registerHttpRoute
    api.registerHttpRoute({
      path: '/my-webhook',
      auth: 'plugin',    // 'plugin' | 'none'
      match: 'exact',    // 'exact' | 'prefix'
      handler: webhookHandler,
    });
    api.registerService(myService);
  },
};

export default plugin;
```

## Plugin API

### OpenClawPluginApi

```typescript
interface OpenClawPluginApi {
  runtime: PluginRuntime;
  config: PluginConfig;
  logger: Logger;
  
  registerChannel(opts: ChannelRegistration): void;
  registerTool(tool: ToolDefinition, opts?: ToolOptions): void;
  registerGatewayMethod(name: string, handler: RpcHandler): void;
  /** @deprecated 已移除。請改用 registerHttpRoute */
  // registerHttpHandler(handler: HttpHandler): void;
  registerHttpRoute(opts: {
    path: string;
    auth: 'plugin' | 'none';
    match: 'exact' | 'prefix';
    handler: HttpHandler;
  }): void;
  registerCli(registrar: CliRegistrar, opts?: CliOptions): void;
  registerService(service: ServiceDefinition): void;
  registerProvider(provider: ProviderDefinition): void;
}

// Plugin Runtime API（外掛可用）
interface PluginRuntime {
  stt: {
    /** 透過 OpenClaw 配置的媒體理解音頻提供者轉錄本地音頻檔案 */
    transcribeAudioFile(filePath: string, opts?: SttOptions): Promise<string>;
  };
  tts: {
    /** 電話通話用 TTS（PCM 音頻 + 取樣率） */
    textToSpeechTelephony(opts: { text: string; cfg: Config }): Promise<TtsResult>;
  };
  system: {
    /** 立即喚醒目標 session 的心跳 */
    requestHeartbeatNow(sessionKey: string): void;
  };
  events: {
    /** 訂閱 agent 事件 */
    onAgentEvent(handler: AgentEventHandler): Unsubscribe;
    /** 訂閱 session transcript 更新（隔離失敗的 listener） */
    onSessionTranscriptUpdate(handler: TranscriptHandler): Unsubscribe;
  };
}
```

### 工具註冊

```typescript
import { Type } from '@sinclair/typebox';

api.registerTool(
  {
    name: 'my_tool',
    description: 'Execute a custom action',
    parameters: Type.Object({
      input: Type.String({ description: 'Input value' }),
      option: Type.Optional(Type.Boolean()),
    }),
    execute: async (toolCallId, params, context) => {
      const { input, option } = params;
      return { result: `Processed: ${input}` };
    },
  },
  { requireApproval: false, category: 'utility' }
);
```

**工具 Schema 注意事項**：
- 避免 `Type.Union`；使用 `stringEnum`/`optionalStringEnum` 代替
- 頂層 schema 必須是 `type: "object"` 搭配 `properties`
- 避免使用 `format` 作為屬性名稱

### Gateway 方法註冊

```typescript
api.registerGatewayMethod(
  'my-plugin.action',
  async ({ params, respond, broadcast }) => {
    try {
      const result = await doSomething(params);
      respond(true, { result });
      broadcast('my-plugin.event', { data: result });
    } catch (error) {
      respond(false, { error: error.message });
    }
  }
);
```

## 依賴管理

### package.json

```json
{
  "name": "@openclaw/my-plugin",
  "version": "1.0.0",
  "type": "module",
  "main": "index.ts",
  "dependencies": {
    "my-sdk": "^1.0.0"
  },
  "devDependencies": {
    "openclaw": "workspace:*"
  },
  "peerDependencies": {
    "openclaw": "*"
  }
}
```

### 重要注意事項

1. **不要在 dependencies 中使用 `workspace:*`** - npm install 會失敗
2. **將 openclaw 放在 devDependencies 或 peerDependencies** - 運行時透過 jiti alias 解析 `openclaw/plugin-sdk`
3. **運行時依賴必須在 dependencies** - 安裝時執行 `npm install --omit=dev`
4. **有 `pnpm.patchedDependencies` 的依賴必須使用精確版本**（無 `^`/`~`）

## 擴充套件開發改善（2026.4.13–2026.4.14）

### Claimable Dedupe Helper

Plugin SDK 新增 `claimable dedupe helper`，提供標準化的去重複基礎設施：

```typescript
import { createClaimableDedupe } from "openclaw/plugin-sdk/dedupe";

// 在 replay handler 中使用
const dedupe = createClaimableDedupe();
const claim = await dedupe.claim(eventId);
if (!claim) return; // 已被其他 handler 處理

try {
  await handleEvent(event);
  await claim.commit(); // 標記成功，不再重試
} catch (error) {
  if (isRetryable(error)) {
    await claim.release(); // 釋放 claim，允許重試
  } else {
    await claim.commit(); // 非可重試錯誤，不重試
  }
}
```

**特性**：
- 序列化競態條件（serialize claimable dedupe races）
- 支援「成功後不再重試，預交付失敗後重試」的語意
- 已在 15+ 頻道（Telegram、Discord、Slack 等）採用

### Context Engine Plugin 改善

自 2026.4.14 起，context engine plugins 支援改善：

- **Per-Iteration Ingest**：引擎從第一個 tool-loop delta 開始進行 compact
- **Idle-Aware 維護**：turn 維護作為 idle-aware 背景工作執行
- **Contract Validation**：`info.id` 與 slot id 不符的 plugin engines 現在快速失敗，防止靜默錯誤

### Plugin 效能最佳實踐

自 2026.4.14 起，以下模式可提升插件效能：

```typescript
// 推薦：懶載入重量級依賴
export async function handleMessage(msg: Message) {
  const { heavyHelper } = await import("./heavy-runtime.ts");
  return heavyHelper.process(msg);
}

// 不推薦：在模組頂層即載入
import { heavyHelper } from "./heavy-runtime.ts"; // ❌ 影響啟動效能
```

核心已在 50+ 路徑採用懶載入模式，插件也應遵循同樣原則。

### Plugin Status / Inspect 改善

- **Context-Engine 能力報告**：`plugins inspect` 正確報告已註冊的 context-engine ID 而非擁有插件 ID（#58766）
- **外部目錄快取**：外部 `preferOver` 目錄查找在每次 auto-enable pass 內快取，大型 `agents.list` 配置不再 CPU 峰值（#66246）

## 擴充套件開發改善（2026.4.18–2026.4.20）

### Plugin Loader Alias 重用（效能改善）

跨重複的同 context 載入重用 plugin loader alias 和 Jiti config resolution（#69316）：
- 減少 import-heavy 測試開銷
- Plugin 啟動時間改善

### Plugin 優先級管理

不同發現的插件共享相同 id 時，只保留最高優先級的 manifest（#41626）：
- 防止低優先級的全域或工作區重複 plugin 在 bundled plugin 旁邊載入
- 影響 Windows multi-workspace 和 custom plugin directory 配置

### Plugin Discovery 跨 Workspace Cache 重用

跨 workspace cache misses 重用 bundled 和 global plugin discovery results（#67940）：
- Windows multi-workspace startup 不再重複做共享同步掃描

### Plugin Webhooks 強化

- 強制同步 plugin 註冊並在失敗時完整回滾
- 快取 SecretRef-backed webhook auth per route
- Plugin startup 和 inbound webhook auth 更確定性

### MCP stdio 環境安全過濾

MCP stdio servers 的 interpreter-startup env keys（如 `NODE_OPTIONS`）被阻止（#69540）：
- 保留普通 credential 和 proxy env vars
- 防止工作區配置的 MCP servers 透過 `NODE_OPTIONS` 等機制影響執行環境

### Detached Task Registration Contract

插件 executors 現在可透過 detached runtime registration contract 管理 detached task lifecycle（#68915）：
- 插件可擁有 detached task lifecycle 和 cancellation
- 不需要觸及 core task internals

### Plugin SDK Secret Input 保留

在發布版本中保留 `secret-input-runtime` function exports：
- Provider plugins 可在 published builds 中讀取 SecretRef-backed setup inputs

### Plugin Memory Capability 保留

read-only snapshot plugin 載入時保留 active memory capability（#69219）：
- Status 和 provider discovery 路徑不再清除 memory public artifacts

### Plugin Bundled Runtime 管理

Bundled plugin runtime 依賴管理大量改善：
- 安裝至插件自己的 runtime 目錄
- 忽略 pnpm 的 `npm_execpath`，跳過 workspace-only package specs
- 重建後重用 source-checkout 修復快取
- 只記錄實際安裝的套件，减少重複 Gateway 啟動時的雜訊

## 擴充套件開發改善（2026.4.15）

### Bundled Runtime Deps 本地化

自 2026.4.15 起，捆綁插件 runtime 依賴本地化至擁有的 extensions（#67099）：
- core 不再攜帶 extension 擁有的 runtime 負擔
- 發布的 tarballs 更精簡
- npm release 驗證若打包的 test cargo 重新出現會失敗

### Plugin SDK 共用快取路徑

多個 Plugin SDK 路徑現在共用 jiti 快取輔助工具（`share*.ts` 系列）：
- Anthropic replay hooks
- Google Gemini provider hook bundle
- OpenAI responses stream/transport hooks
- MiniMax provider hook bundle
- 各種 loader jiti cache 路徑

好處：減少重複的 jiti 載入開銷，改善插件啟動時間。

### Plugin Source Alias 修復

擴寬各種 plugin SDK 來源別名候選（#66649 系列）：
- `plugin-sdk` root alias 候選擴寬
- `extension-api` source alias 候選擴寬
- Matrix wrapper scoped SDK aliases 對齊
- safe root alias subpaths 按雜湊值排序（確保 prompt cache 穩定性）

### Setup Entry Contract

支援 loader 中的捆綁 setup-entry contract（#66261）：
- 保留分割插件 + secrets 入口點的 channel secret contract
- setup-mode flows 繼續取得 channel secret contract

### Packaging / Test Cargo 裁剪

- 從捆綁插件 runtime 依賴中裁剪常見的 test/spec cargo（#67275）
- `pnpm build` 驗證打包 test cargo 不再重新出現
- 發布的 tarballs 更精簡，核心不再攜帶 extension-owned runtime 負擔

### Embedding Provider for Memory Search

插件可使用 `github-copilot` embedding provider 進行記憶搜尋（#61718）：

```typescript
// 在插件中使用 Copilot embedding provider
import { createCopilotEmbeddingHelper } from "openclaw/plugin-sdk/github-copilot";

const helper = createCopilotEmbeddingHelper({
  remoteOverride: config.remoteOverride,
});
const embeddings = await helper.embed(texts);
```
