# 擴充套件開發

## 概述

OpenClaw 擴充套件（插件）是獨立的模組，可新增頻道、工具、CLI 命令等功能。本文件說明如何開發和發布擴充套件。

## 擴充套件結構

### 目錄結構

```
extensions/<plugin-id>/
├── openclaw.plugin.json   # 插件元資料
├── index.ts               # 入口檔案
├── package.json           # 依賴管理
├── README.md              # 說明文件
├── CHANGELOG.md           # 變更記錄
└── src/                   # 原始碼
    ├── channel.ts         # 頻道實作（如適用）
    ├── runtime.ts         # 運行時設定
    ├── monitor.ts         # 訊息監控
    ├── send.ts            # 發送邏輯
    ├── onboarding.ts      # 引導流程
    ├── tool.ts            # 工具實作（如適用）
    └── types.ts           # 類型定義
```

### 元資料檔案

```json
// openclaw.plugin.json
{
  "id": "my-plugin",
  "name": "My Plugin",
  "version": "1.0.0",
  "description": "A custom plugin for OpenClaw",
  
  "channels": ["my-channel"],
  "providers": ["my-provider"],
  "kind": "channel",  // channel | memory | tool | provider
  
  "configSchema": {
    "type": "object",
    "properties": {
      "apiKey": {
        "type": "string",
        "description": "API Key for the service"
      },
      "enabled": {
        "type": "boolean",
        "default": true
      }
    },
    "required": ["apiKey"]
  }
}
```

### 入口檔案

```typescript
// index.ts
import type { OpenClawPluginApi, OpenClawPlugin } from 'openclaw/plugin-sdk';
import { emptyPluginConfigSchema } from 'openclaw/plugin-sdk';
import { channelPlugin } from './src/channel.js';
import { setRuntime } from './src/runtime.js';

const plugin: OpenClawPlugin = {
  id: 'my-plugin',
  name: 'My Plugin',
  description: 'A custom plugin',
  configSchema: emptyPluginConfigSchema(),
  
  register(api: OpenClawPluginApi) {
    setRuntime(api.runtime);
    api.registerChannel({ plugin: channelPlugin });
    api.registerTool(myTool);
    api.registerCli(registerMyCli);
    api.registerGatewayMethod('my-plugin.action', myHandler);
    // ⚠️ registerHttpHandler 已移除，請改用 registerHttpRoute
    api.registerHttpRoute({
      path: '/my-webhook',
      auth: 'plugin',    // 'plugin' | 'none'
      match: 'exact',    // 'exact' | 'prefix'
      handler: webhookHandler,
    });
    api.registerService(myService);
  },
};

export default plugin;
```

## Plugin API

### OpenClawPluginApi

```typescript
interface OpenClawPluginApi {
  runtime: PluginRuntime;
  config: PluginConfig;
  logger: Logger;
  
  registerChannel(opts: ChannelRegistration): void;
  registerTool(tool: ToolDefinition, opts?: ToolOptions): void;
  registerGatewayMethod(name: string, handler: RpcHandler): void;
  /** @deprecated 已移除。請改用 registerHttpRoute */
  // registerHttpHandler(handler: HttpHandler): void;
  registerHttpRoute(opts: {
    path: string;
    auth: 'plugin' | 'none';
    match: 'exact' | 'prefix';
    handler: HttpHandler;
  }): void;
  registerCli(registrar: CliRegistrar, opts?: CliOptions): void;
  registerService(service: ServiceDefinition): void;
  registerProvider(provider: ProviderDefinition): void;
}

// Plugin Runtime API（外掛可用）
interface PluginRuntime {
  stt: {
    /** 透過 OpenClaw 配置的媒體理解音頻提供者轉錄本地音頻檔案 */
    transcribeAudioFile(filePath: string, opts?: SttOptions): Promise<string>;
  };
  tts: {
    /** 電話通話用 TTS（PCM 音頻 + 取樣率） */
    textToSpeechTelephony(opts: { text: string; cfg: Config }): Promise<TtsResult>;
  };
  system: {
    /** 立即喚醒目標 session 的心跳 */
    requestHeartbeatNow(sessionKey: string): void;
  };
  events: {
    /** 訂閱 agent 事件 */
    onAgentEvent(handler: AgentEventHandler): Unsubscribe;
    /** 訂閱 session transcript 更新（隔離失敗的 listener） */
    onSessionTranscriptUpdate(handler: TranscriptHandler): Unsubscribe;
  };
}
```

### Plugin SDK 匯出與子路徑

自 2026.3.3 起，推薦使用 SDK 子路徑匯入。自 2026.3.9 起，所有 30+ 個官方外掛已全面遷移至 scoped subpath imports：

```typescript
// 通用外掛 API（推薦）
import type { OpenClawPlugin, OpenClawPluginApi } from 'openclaw/plugin-sdk/core';

// 頻道專屬外掛 SDK
import { ... } from 'openclaw/plugin-sdk/channel';
import { ... } from 'openclaw/plugin-sdk/telegram';
import { ... } from 'openclaw/plugin-sdk/discord';
import { ... } from 'openclaw/plugin-sdk/slack';
import { ... } from 'openclaw/plugin-sdk/signal';
import { ... } from 'openclaw/plugin-sdk/imessage';
import { ... } from 'openclaw/plugin-sdk/whatsapp';
import { ... } from 'openclaw/plugin-sdk/line';

// 工具外掛 SDK
import { ... } from 'openclaw/plugin-sdk/allowlist';
import { ... } from 'openclaw/plugin-sdk/resolve-target';
```

> **相容性**：`openclaw/plugin-sdk` 根匯入仍然支援（透過 root-alias shim lazily load），確保現有外部外掛不受影響。新/遷移的外掛應使用 scoped subpath imports 以獲得更好的 tree-shaking 和載入效能。

```typescript
// 設備配對 API
import {
  approveDevicePairing,
  listDevicePairing,
  rejectDevicePairing,
} from 'openclaw/plugin-sdk';

// 頻道目錄與狀態問題收集
import {
  collectDiscordDirectoryEntries,
  collectTelegramStatusIssues,
  collectSlackStatusIssues,
  // ... 各頻道對應函數
} from 'openclaw/plugin-sdk';

// Allowlist 解析工具
import {
  resolveDiscordAllowlist,
  resolveTelegramAllowlist,
  // ...
} from 'openclaw/plugin-sdk';
```

### 運行時服務

```typescript
interface PluginRuntime {
  tts: TTSService;
  tools: ToolRegistry;
  channels: ChannelManager;
  stt: SttService;
  system: SystemService;    // requestHeartbeatNow(...)
  events: EventsService;    // onAgentEvent, onSessionTranscriptUpdate

  getConfig<T>(key: string): T | undefined;
  setConfig<T>(key: string, value: T): void;

  emit(event: string, payload: unknown): void;
  on(event: string, handler: EventHandler): void;
}
```

### channelRuntime（頻道外掛）

`ChannelGatewayContext` 上暴露 `channelRuntime`，讓外部頻道外掛可存取共用的 runtime helpers（reply/routing/session/text/media/commands），無需內部匯入。

## 註冊類型

### 頻道註冊

```typescript
api.registerChannel({
  plugin: {
    id: 'my-channel',
    name: 'My Channel',
    async initialize(config) { /* ... */ },
    async startMonitor(handler) { /* ... */ },
    async sendMessage(chatId, message) { /* ... */ },
    async getStatus() { return { connected: true }; },
    async shutdown() { /* ... */ },
  },
  dock: { icon: 'message-circle', color: '#007AFF' },
});
```

### 工具註冊

```typescript
import { Type } from '@sinclair/typebox';

api.registerTool(
  {
    name: 'my_tool',
    description: 'Execute a custom action',
    parameters: Type.Object({
      input: Type.String({ description: 'Input value' }),
      option: Type.Optional(Type.Boolean()),
    }),
    execute: async (toolCallId, params, context) => {
      const { input, option } = params;
      return { result: `Processed: ${input}` };
    },
  },
  { requireApproval: false, category: 'utility' }
);
```

**工具 Schema 注意事項**：
- 避免 `Type.Union`；使用 `stringEnum`/`optionalStringEnum` 代替
- 頂層 schema 必須是 `type: "object"` 搭配 `properties`
- 避免使用 `format` 作為屬性名稱

### Gateway 方法註冊

```typescript
api.registerGatewayMethod(
  'my-plugin.action',
  async ({ params, respond, broadcast }) => {
    try {
      const result = await doSomething(params);
      respond(true, { result });
      broadcast('my-plugin.event', { data: result });
    } catch (error) {
      respond(false, { error: error.message });
    }
  }
);
```

### HTTP 處理器註冊

```typescript
api.registerHttpHandler({
  path: '/webhooks/my-plugin',
  method: 'POST',
  handler: async (req, res) => {
    const payload = req.body;
    await processWebhook(payload);
    res.json({ success: true });
  },
});
```

## 依賴管理

### package.json

```json
{
  "name": "@openclaw/my-plugin",
  "version": "1.0.0",
  "type": "module",
  "main": "index.ts",
  "dependencies": {
    "my-sdk": "^1.0.0"
  },
  "devDependencies": {
    "openclaw": "workspace:*"
  },
  "peerDependencies": {
    "openclaw": "*"
  }
}
```

### 重要注意事項

1. **不要在 dependencies 中使用 `workspace:*`** - npm install 會失敗
2. **將 openclaw 放在 devDependencies 或 peerDependencies** - 運行時透過 jiti alias 解析 `openclaw/plugin-sdk`
3. **運行時依賴必須在 dependencies** - 安裝時執行 `npm install --omit=dev`
4. **有 `pnpm.patchedDependencies` 的依賴必須使用精確版本**（無 `^`/`~`）

## 發布

### 本地安裝

```bash
# 從 extensions 目錄安裝
openclaw plugins install ./extensions/my-plugin

# 或在配置中啟用
# ~/.openclaw/openclaw.json
{
  "plugins": {
    "entries": {
      "my-plugin": {
        "enabled": true,
        "path": "~/path/to/my-plugin"
      }
    }
  }
}
```

### 發布到 ClawHub / npm

自 2026.3.23 起，插件安裝預設從 **ClawHub（marketplace）優先**，僅在 ClawHub 找不到時才 fallback 至 npm：

```bash
# 安裝插件（ClawHub 優先，npm fallback）
openclaw plugins install my-plugin

# 明確從 npm 安裝
openclaw plugins install @openclaw/my-plugin

# Slash 快捷安裝（2026.3.23+）
/install my-plugin
```

### 發布到 npm

```bash
cd extensions/my-plugin
npm publish --access public
```

## 擴充套件類型範例

### device-pair（設備配對）

配置選項：`publicUrl`（Gateway WebSocket URL）。

### phone-control（手機控制）

控制高風險手機節點命令（相機/螢幕/寫入），支援自動過期。

### voice-call（語音通話）

支援 Twilio、Telnyx、Plivo 語音通話整合。

### Synology Chat

原生 Synology Chat 頻道插件。

### diffs（差異渲染）

新的可選 `diffs` 插件工具，提供唯讀差異渲染功能：
- 支援 before/after 文字或 unified patch
- Gateway viewer URL 用於 canvas 和 PNG 圖片輸出

### acpx（ACPX 串流）

ACPX 插件支援（pinned to `0.1.16`），提供可配置的 ACPX 命令/版本偵測，以及簡化的 ACP 串流交付（`final_only` 預設）。

### context-engine（自訂上下文管理）

Context Engine 外掛類型，允許外掛註冊自訂上下文管理策略：

```json
// openclaw.plugin.json
{
  "kind": "context-engine",
  "id": "my-context-engine",
  // ...
}
```

外掛可透過 `api.runtime.modelAuth` 存取 model auth API。使用 slash-delimited schema lookup paths 進行配置。

## 最佳實踐

1. **錯誤處理**: 妥善處理所有可能的錯誤情況
2. **日誌記錄**: 使用 `api.logger` 記錄重要操作
3. **配置驗證**: 定義完整的 configSchema
4. **資源清理**: 在 shutdown 時清理所有資源
5. **類型安全**: 使用 TypeScript 和 TypeBox
6. **文件齊全**: 提供 README 和 CHANGELOG
7. **測試覆蓋**: 撰寫單元測試和整合測試
8. **安全掃描**: 程式碼會被安全掃描器檢查，避免使用 eval、child_process 直接呼叫等
9. **標籤同步**: 新增擴充套件時，更新 `.github/labeler.yml` 並建立 GitHub 標籤
10. **Windows 相容**: 避免直接 spawn `.cmd`；使用 `node` + npm CLI 腳本解析以避免 `EINVAL`
11. **插件優先序**: 綁定插件優先於自動發現的全域擴充套件載入；`plugins.load.paths` 最高優先
12. **SDK 子路徑**: 新外掛應使用 `openclaw/plugin-sdk/core` 或頻道專屬子路徑
13. **Hook 事件**: `session_start`/`session_end` 現包含 `sessionKey` 供生命週期關聯
14. **懶加載邊界**: 外掛啟動匯入拆分為 `openclaw/plugin-sdk/core` + 頻道專屬，減少啟動負擔
15. **Context Engine**: 新增 `kind: "context-engine"` manifest 類型，支援自訂 context management
16. **Model Auth API**: context-engine 外掛可透過 `api.runtime.modelAuth` 存取 model authentication
17. **Prompt Hook Policy**: 外掛 prompt hooks 加入 runtime policy 驗證
18. **Plugin 快取清除**: 安裝後自動清除 discovery cache
19. **Integrity Drift**: unpinned 更新時避免 false integrity drift 提示
20. **工作區插件信任**（2026.3.12+）: 工作區插件不再自動載入，需透過 `plugins.allow` 明確信任（GHSA-99qw-6mr3-36qr）
21. **Plugin 碰撞偵測**（2026.3.13+）: channel 和 binding 碰撞載入時 fail fast
22. **Provider Plugin**: Ollama/vLLM/SGLang 已遷移至 provider-plugin 架構，provider 擁有 onboarding、discovery、model-picker 和 post-selection hooks
23. **Plugin-SDK 記憶體修復**: subpath entries 在共用 build pass 中 bundle，避免重複 chunks 和記憶體膨脹
24. **ClawHub 優先安裝**（2026.3.23+）: 插件安裝預設從 ClawHub 取得，npm 為 fallback
25. **Slash 安裝**（2026.3.23+）: 支援 `/install <plugin>` 快捷安裝
26. **Plugin Capability 系統**（2026.3.23+）: Web Search、Image Generation、TTS/Speech、Media Understanding 透過 capability 註冊
27. **Plugin SDK Facades 自動生成**（2026.3.31）: 所有 extension 透過自動生成的 public barrel facades 路由
28. **Legacy Compat 棄用**（2026.3.31）: 舊版 provider compat subpaths 和 channel-runtime 相容層已棄用，發出遷移警告
29. **before_install Hook**（2026.3.31）: 新增 `before_install` hook 用於安裝時掃描器
30. **requireApproval Hook**（2026.3.31）: `before_tool_call` hook 支援非同步 `requireApproval`
31. **runId 暴露**（2026.3.31）: agent hook context 中暴露 `runId`
32. **Inbound Attachments**（2026.3.31）: hooks 傳遞 inbound attachment arrays 給插件
33. **Skills/Plugins Install Gate**（2026.3.31）: 安裝時 `critical` 危險碼掃描結果 fail closed，需 `--dangerously-force-unsafe-install` 明確覆蓋
34. **Provider Runtime Plugin 化**（2026.3.31）: provider runtime 完全遷入 extension packages
35. **Channel Runtime 抽象**（2026.3.31）: 所有核心頻道遷移到統一的 channel runtime 抽象
36. **Multi-kind Plugins**（2026.3.31）: 支援 multi-kind plugins for dual slot ownership
37. **Channel Uninstall Cleanup**（2026.3.31）: 解除安裝時清理 channel config
