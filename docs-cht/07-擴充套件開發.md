# 擴充套件開發

## 概述

OpenClaw 擴充套件（插件）是獨立的模組，可新增頻道、工具、CLI 命令等功能。本文件說明如何開發和發布擴充套件。

## 擴充套件結構

### 目錄結構

```
extensions/<plugin-id>/
├── openclaw.plugin.json   # 插件元資料
├── index.ts               # 入口檔案
├── package.json           # 依賴管理
├── README.md              # 說明文件
├── CHANGELOG.md           # 變更記錄
└── src/                   # 原始碼
    ├── channel.ts         # 頻道實作（如適用）
    ├── runtime.ts         # 運行時設定
    ├── monitor.ts         # 訊息監控
    ├── send.ts            # 發送邏輯
    ├── onboarding.ts      # 引導流程
    ├── tool.ts            # 工具實作（如適用）
    └── types.ts           # 類型定義
```

### 元資料檔案

```json
// openclaw.plugin.json
{
  "id": "my-plugin",
  "name": "My Plugin",
  "version": "1.0.0",
  "description": "A custom plugin for OpenClaw",
  
  "channels": ["my-channel"],
  "providers": ["my-provider"],
  "kind": "channel",  // channel | memory | tool | provider
  
  "configSchema": {
    "type": "object",
    "properties": {
      "apiKey": {
        "type": "string",
        "description": "API Key for the service"
      },
      "enabled": {
        "type": "boolean",
        "default": true
      }
    },
    "required": ["apiKey"]
  }
}
```

### 入口檔案

```typescript
// index.ts
import type { OpenClawPluginApi, OpenClawPlugin } from 'openclaw/plugin-sdk';
import { emptyPluginConfigSchema } from 'openclaw/plugin-sdk';
import { channelPlugin } from './src/channel.js';
import { setRuntime } from './src/runtime.js';

const plugin: OpenClawPlugin = {
  id: 'my-plugin',
  name: 'My Plugin',
  description: 'A custom plugin',
  configSchema: emptyPluginConfigSchema(),
  
  register(api: OpenClawPluginApi) {
    setRuntime(api.runtime);
    api.registerChannel({ plugin: channelPlugin });
    api.registerTool(myTool);
    api.registerCli(registerMyCli);
    api.registerGatewayMethod('my-plugin.action', myHandler);
    api.registerHttpHandler(webhookHandler);
    api.registerService(myService);
  },
};

export default plugin;
```

## Plugin API

### OpenClawPluginApi

```typescript
interface OpenClawPluginApi {
  runtime: PluginRuntime;
  config: PluginConfig;
  logger: Logger;

  registerCli(registrar: CliRegistrar, opts?: CliOptions): void;
  registerService(service: ServiceDefinition): void;
  registerProvider(provider: ProviderDefinition): void;
}
```

## 註冊類型

### 頻道註冊

```typescript
api.registerChannel({
  plugin: {
    id: 'my-channel',
    name: 'My Channel',
    async initialize(config) { /* ... */ },
    async startMonitor(handler) { /* ... */ },
    async sendMessage(chatId, message) { /* ... */ },
    async getStatus() { return { connected: true }; },
    async shutdown() { /* ... */ },
  },
  dock: { icon: 'message-circle', color: '#007AFF' },
});
```

### 工具註冊

```typescript
import { Type } from '@sinclair/typebox';

api.registerTool(
  {
    name: 'my_tool',
    description: 'Execute a custom action',
    parameters: Type.Object({
      input: Type.String({ description: 'Input value' }),
      option: Type.Optional(Type.Boolean()),
    }),
    execute: async (toolCallId, params, context) => {
      const { input, option } = params;
      return { result: `Processed: ${input}` };
    },
  },
  { requireApproval: false, category: 'utility' }
);
```

**工具 Schema 注意事項**：
- 避免 `Type.Union`；使用 `stringEnum`/`optionalStringEnum` 代替
- 頂層 schema 必須是 `type: "object"` 搭配 `properties`
- 避免使用 `format` 作為屬性名稱

### Gateway 方法註冊

```typescript
api.registerGatewayMethod(
  'my-plugin.action',
  async ({ params, respond, broadcast }) => {
    try {
      const result = await doSomething(params);
      respond(true, { result });
      broadcast('my-plugin.event', { data: result });
    } catch (error) {
      respond(false, { error: error.message });
    }
  }
);
```

### HTTP 處理器註冊

```typescript
api.registerHttpHandler({
  path: '/webhooks/my-plugin',
  method: 'POST',
  handler: async (req, res) => {
    const payload = req.body;
    await processWebhook(payload);
    res.json({ success: true });
  },
});
```

### CLI 命令註冊

```typescript
api.registerCli(

```

## 依賴管理

### package.json

```json
{
  "name": "@openclaw/my-plugin",
  "version": "1.0.0",
  "type": "module",
  "main": "index.ts",
  "dependencies": {
    "my-sdk": "^1.0.0"
  },
  "devDependencies": {
    "openclaw": "workspace:*"
  },
  "peerDependencies": {
    "openclaw": "*"
  }
}
```

### 重要注意事項

1. **不要在 dependencies 中使用 `workspace:*`** - npm install 會失敗
2. **將 openclaw 放在 devDependencies 或 peerDependencies** - 運行時透過 jiti alias 解析 `openclaw/plugin-sdk`
3. **運行時依賴必須在 dependencies** - 安裝時執行 `npm install --omit=dev`
4. **有 `pnpm.patchedDependencies` 的依賴必須使用精確版本**（無 `^`/`~`）

## 發布

### 本地安裝

```bash
# 從 extensions 目錄安裝
openclaw plugins install ./extensions/my-plugin

# 或在配置中啟用
# ~/.openclaw/openclaw.json
{
  "plugins": {
    "entries": {
      "my-plugin": {
        "enabled": true,
        "path": "~/path/to/my-plugin"
      }
    }
  }
}
```

### 發布到 npm

```bash
cd extensions/my-plugin
npm publish --access public
```

### 從 npm 安裝

```bash
openclaw plugins install @openclaw/my-plugin
```

## 最佳實踐

1. **錯誤處理**: 妥善處理所有可能的錯誤情況
2. **日誌記錄**: 使用 `api.logger` 記錄重要操作
3. **配置驗證**: 定義完整的 configSchema
4. **資源清理**: 在 shutdown 時清理所有資源
5. **類型安全**: 使用 TypeScript 和 TypeBox
6. **文件齊全**: 提供 README 和 CHANGELOG
7. **測試覆蓋**: 撰寫單元測試和整合測試
8. **安全掃描**: 程式碼會被安全掃描器檢查，避免使用 eval、child_process 直接呼叫等
9. **標籤同步**: 新增擴充套件時，更新 `.github/labeler.yml` 並建立 GitHub 標籤
