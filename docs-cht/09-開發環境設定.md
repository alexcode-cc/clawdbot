# 開發環境設定

## 系統需求

### 必要條件

| 項目 | 版本需求 |
|------|---------|
| Node.js | >= 22.16.0 |
| pnpm | 10.23.0 |
| Git | 最新版本 |

### 選擇性（依開發目標）

| 項目 | 用途 |
|------|------|
| Bun | TypeScript 執行（推薦） |
| Xcode | iOS/macOS 應用開發 |
| Android Studio | Android 應用開發 |
| Docker | 容器化測試 |

## 快速開始

### 1. 複製專案

```bash
git clone https://github.com/openclaw/openclaw.git
cd openclaw
```

### 2. 安裝依賴

```bash
# 使用 pnpm（主要）
pnpm install

# 或使用 Bun（也支援）
bun install
```

### 3. 建置專案

```bash
pnpm build
```

### 4. 執行測試

```bash
pnpm test
```

### 5. 安裝 Git Hooks

```bash
prek install
```

## 開發命令

### CLI 開發

```bash
# 執行 CLI（開發模式，使用 Bun）
pnpm openclaw --help
pnpm openclaw config list
pnpm openclaw agent

# 或使用 npm scripts
pnpm dev
```

### Gateway 開發

```bash
# 啟動 Gateway（開發模式）
pnpm gateway:dev

# 監視模式（自動重載）
pnpm gateway:watch
```

### UI 開發

```bash
# 安裝 UI 依賴
pnpm ui:install

# 開發模式
pnpm ui:dev

# 建置
pnpm ui:build
```

### TUI 開發

```bash
# 啟動 TUI
pnpm tui

# 開發模式
pnpm tui:dev
```

## 程式碼品質

### Lint 與格式檢查

```bash
# 執行 lint + 格式檢查（Oxlint + Oxfmt）
pnpm check

# 格式檢查
pnpm format

# 格式自動修復
pnpm format:fix
```

### 程式碼風格

- TypeScript (ESM)，嚴格類型，避免 `any`
- 不要加 `@ts-nocheck`，不要停用 `no-explicit-any`
- 不要透過 prototype mutation 共享行為
- 檔案盡量控制在 500 行以內
- 使用 `src/terminal/palette.ts` 共享色彩（不要硬編碼顏色）
- 動態匯入護欄：不要混合 `await import("x")` 和靜態 `import ... from "x"`；使用 `.runtime.ts` 懶加載邊界
- 建置後檢查 `[INEFFECTIVE_DYNAMIC_IMPORT]` 警告

## 測試

### 單元測試

```bash
# 執行所有測試
pnpm test

# 監視模式
pnpm test:watch

# 覆蓋率報告（70% 門檻）
pnpm test:coverage

# 記憶體壓力環境（non-Mac-Studio hosts）
OPENCLAW_TEST_PROFILE=low OPENCLAW_TEST_SERIAL_GATEWAY=1 pnpm test
```

### E2E 測試

```bash
# E2E 測試
pnpm test:e2e

# 實時測試（需要真實 API 金鑰）
CLAWDBOT_LIVE_TEST=1 pnpm test:live

# 包含 provider 實時測試
LIVE=1 pnpm test:live
```

### Docker 測試

```bash
# 模型實時測試
pnpm test:docker:live-models

# Gateway 實時測試
pnpm test:docker:live-gateway

# Onboarding 測試
pnpm test:docker:onboard

# 插件測試
pnpm test:docker:plugins
```

### Docker 映像檔

- 官方映像：`ghcr.io/openclaw/openclaw`（標籤：`main`、`latest`、`<version>`）
- Dockerfile 內建 `HEALTHCHECK`（`GET /healthz`）
- OCI 基底映像標籤和 source/documentation/license 元資料
- 擴充套件目錄權限：`755`（目錄）、`644`（檔案）

### 安裝腳本測試

```bash
# Smoke 測試
pnpm test:install:smoke

# 非 root 路徑
OPENCLAW_INSTALL_SMOKE_SKIP_NONROOT=1 pnpm test:install:smoke
```

## 文件開發

### 本地預覽

```bash
# 安裝 Mintlify CLI
npm i -g mintlify

# 啟動開發伺服器
pnpm docs:dev
```

### 文件規範

- 託管於 Mintlify (`docs.openclaw.ai`)
- 內部連結：根相對路徑，不加 `.md`/`.mdx`（例：`[Config](/configuration)`）
- 章節交叉引用用錨點（例：`[Hooks](/configuration#hooks)`）
- 避免在標題中使用 em dash 和撇號（會破壞 Mintlify 錨點連結）

## 環境變數

### 開發用環境變數

| 變數 | 說明 |
|------|------|
| `CLAWDBOT_CONFIG_PATH` | 配置檔路徑 |
| `CLAWDBOT_PROFILE` | 配置檔案名稱（如 `dev`） |
| `CLAWDBOT_LOG_LEVEL` | 日誌等級（debug, info, warn, error） |
| `CLAWDBOT_SKIP_CHANNELS` | 跳過頻道載入（加速開發） |
| `CLAWDBOT_LIVE_TEST` | 啟用實時測試 |
| `CLAWDBOT_DEBUG` | 除錯模式 |
| `OLLAMA_API_KEY` | Ollama 認證（任意值即可） |
| `TOGETHER_API_KEY` | Together AI API 金鑰 |
| `OPENCLAW_SHELL` | 自動注入，標示 OpenClaw shell 執行環境 |
| `OPENCLAW_GATEWAY_TOKEN` | Gateway 認證 token |
| `OPENCLAW_ALLOW_INSECURE_PRIVATE_WS` | 允許私網 `ws://` 連線（`1`） |
| `OPENCLAW_BROWSER_NO_SANDBOX` | 沙箱瀏覽器容器中停用 sandbox |

## 故障排除

### 常見問題

#### pnpm install 失敗

```bash
pnpm store prune
rm -rf node_modules pnpm-lock.yaml
pnpm install
```

#### 缺少依賴（vitest not found 等）

先執行 `pnpm install`，再重新執行原始命令。

## 版本發布

### 發布檢查清單

```bash
# 執行發布檢查
node --import tsx scripts/release-check.ts
pnpm release:check
pnpm test:install:smoke
```

詳細流程請參閱 `docs/reference/RELEASING.md` 和 `docs/platforms/mac/release.md`。

| 檔案 | 位置 |
|------|------|

| macOS | `apps/macos/Sources/OpenClaw/Resources/Info.plist` |
