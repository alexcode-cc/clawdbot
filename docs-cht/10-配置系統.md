# 配置系統

## 概述

OpenClaw 使用 JSON5 格式的配置檔案，支援環境變數替換、包含指令、Zod Schema 驗證等功能。配置採用嚴格驗證——未知鍵會導致 Gateway 拒絕啟動。

## 配置檔案位置

### 預設路徑

```
~/.openclaw/openclaw.json
```

### 自訂路徑

```bash
# 環境變數
export CLAWDBOT_CONFIG_PATH=/path/to/config.json5

# 命令列參數
openclaw --config /path/to/config.json5
```

### 配置檔案（Profile）

```bash
# 使用 dev 檔案
export CLAWDBOT_PROFILE=dev
# 會載入 ~/.openclaw/openclaw.dev.json

# 或命令列
openclaw --profile dev
```

### 模型失效容錯備份（Model Failover / Fallback）

OpenClaw 實作了**兩層容錯系統**：先在同一提供者的 Auth Profile 之間輪替，再依序切換至備份模型。

#### 基本設定

```json5
{
  agents: {
    defaults: {
      // 方式 1：僅指定主要模型（無備份）
      model: "anthropic/claude-sonnet-4-6",

      // 方式 2：主要 + 備份鏈
      model: {
        primary: "anthropic/claude-sonnet-4-6",
        fallbacks: [
          "openai/gpt-5.2",                          // 第一備份
          "google-antigravity/gemini-2-flash",        // 第二備份
          "ollama/qwen3.5:27b",                       // 第三備份（本地）
        ],
      },

      // 模型目錄（別名與設定）
      models: {
        "anthropic/claude-sonnet-4-6": { alias: "Sonnet" },
        "openai/gpt-5.2": { alias: "GPT-5" },
        "google-antigravity/gemini-2-flash": { alias: "Gemini" },
      },
    },
  },
}
```

#### Per-Agent 覆寫

個別 Agent 可覆寫預設的 fallback 鏈：

```json5
{
  agents: {
    defaults: {
      model: {
        primary: "anthropic/claude-sonnet-4-6",
        fallbacks: ["openai/gpt-5.2"],
      },
    },
    list: [
      {
        id: "code-agent",
        model: {
          primary: "anthropic/claude-opus-4-6",      // 覆寫主要模型
          fallbacks: ["anthropic/claude-sonnet-4-6"], // 覆寫 fallback
        },
      },
      {
        id: "chat-agent",
        model: "ollama/qwen3.5:27b",                 // 僅主要模型，無 fallback
      },
    ],
  },
}
```

#### 觸發 Fallback 的條件

系統將錯誤分為 **9 種失敗原因**，每種有不同的處理方式：

| 失敗原因 | 觸發條件 | 處理方式 |
|---------|---------|---------|
| **billing** | 額度不足、402 Payment Required（永久性） | Profile 停用（5-24h 指數退避） |
| **rate_limit** | 429 Too Many Requests、配額超限、TPM/RPM 限制 | Profile 冷卻（1min-1h 指數退避） |
| **overloaded** | 503 服務過載、高需求提示 | Profile 冷卻（1min-1h） |
| **auth** | API Key 無效、Token 過期、401/403 | Profile 冷卻（1min-1h） |
| **auth_permanent** | API Key 撤銷、帳號停用 | Profile 停用（5-24h） |
| **timeout** | 連線逾時、ECONNRESET、502/504、socket hang up | Profile 冷卻（1min-1h） |
| **model_not_found** | 模型不存在（404） | 直接嘗試下一個 fallback 模型 |
| **format** | 請求格式錯誤（400/422） | 不觸發 fallback（非暫時性） |
| **session_expired** | Session 已失效（410） | 嘗試 fallback 但不可重試 |

> **注意**：Context overflow（上下文溢出）**不觸發** fallback，而是立即觸發 compaction（壓縮）。

#### 冷卻時間設定

```json5
{
  auth: {
    cooldowns: {
      // Billing 停用初始退避時間（小時）
      billingBackoffHours: 5,         // 預設 5h

      // Per-provider Billing 退避覆寫
      billingBackoffHoursByProvider: {
        anthropic: 2,                 // Anthropic 2h
        openai: 8,                    // OpenAI 8h
        ollama: 1,                    // Ollama 1h
      },

      // Billing 停用最大時間（小時）
      billingMaxHours: 24,            // 預設 24h 上限

      // 失敗計數視窗（小時）
      failureWindowHours: 24,         // 24h 內無失敗則重設計數
    },
  },
}
```

#### 冷卻退避算法

**暫時性失敗（rate_limit / timeout / overloaded）— 指數退避**：

| 失敗次數 | 冷卻時間 |
|---------|---------|
| 第 1 次 | 1 分鐘 |
| 第 2 次 | 5 分鐘 |
| 第 3 次 | 25 分鐘 |
| 第 4+ 次 | 1 小時（上限） |

**Billing / Auth Permanent 失敗 — 雙倍退避**：

| 失敗次數 | 停用時間（預設） |
|---------|---------------|
| 第 1 次 | 5 小時 |
| 第 2 次 | 10 小時 |
| 第 3 次 | 20 小時 |
| 第 4+ 次 | 24 小時（上限） |

冷卻到期後自動重設失敗計數，下次失敗從第 1 次重新開始（斷路器半開→關閉）。

#### HTTP 402 智慧分類

HTTP 402（Payment Required）會進一步分析錯誤訊息內容：

| 分類為 rate_limit（暫時性） | 分類為 billing（永久性） |
|---------------------------|----------------------|
| 提及 daily/weekly/monthly limit + reset | insufficient credits/quota |
| spending limit + organization scoped | upgrade plan, current plan limit |
| automatic quota refresh | hard limit reached |
| rolling time window | credit balance 不足 |

#### Probe（探測）機制

當所有 Auth Profile 都在冷卻中時，系統可以在限定條件下**探測**（嘗試使用冷卻中的 Profile）：

| 參數 | 值 | 說明 |
|------|-----|------|
| 探測間隔 | 30 秒 | 同一 provider 的最小探測間隔 |
| 到期前探測 | 2 分鐘 | 冷卻到期前 2 分鐘開始探測 |
| 探測狀態 TTL | 24 小時 | 探測狀態過期時間 |
| 追蹤上限 | 256 | 記憶體中追蹤的 provider 數量上限 |

**探測規則**：
- **主要模型**：每 30 秒允許一次探測
- **同 provider fallback**：暫時性失敗（rate_limit/overloaded/unknown）才允許探測，每次 run 最多一次
- **不同 provider fallback**：不需探測，直接按順序嘗試

#### Fallback 決策流程

```
對每個候選模型（主要 → fallback1 → fallback2 → ...）：

1. 檢查 provider 的 Auth Profile 是否在冷卻中
   ├─ 是：
   │  ├─ auth / auth_permanent → 跳過此候選
   │  ├─ billing → 有其他 fallback 時跳過，接近到期時探測
   │  └─ rate_limit / overloaded → 允許受限探測
   └─ 否 → 正常嘗試

2. 嘗試使用此候選模型
   ├─ 成功 → 返回結果
   ├─ Context overflow → 立即拋出（觸發 compaction，不繼續 fallback）
   └─ Failover error → 記錄，繼續下一個候選

3. 所有候選都失敗 → 拋出摘要錯誤
```

#### 繞過冷卻的提供者

以下提供者**永遠不會被放入冷卻**（因為它們是聚合路由器，錯誤由上游 provider 產生）：
- `openrouter`
- `kilocode`

#### 狀態儲存

Auth Profile 的使用狀態儲存在：`~/.openclaw/agents/<agentId>/agent/auth-profiles.json`

記錄的資訊包括：
- `lastUsed` — 最後成功使用時間
- `cooldownUntil` — 暫時性冷卻到期時間
- `disabledUntil` — Billing/Auth 停用到期時間
- `disabledReason` — 停用原因
- `errorCount` — 當前視窗內的失敗次數
- `failureCounts` — 按原因分類的失敗次數
- `lastFailureAt` — 最後失敗時間

#### 完整設定範例

```json5
{
  agents: {
    defaults: {
      model: {
        primary: "anthropic/claude-sonnet-4-6",
        fallbacks: [
          "openai/gpt-5.2",
          "google-antigravity/gemini-2-flash",
        ],
      },
      models: {
        "anthropic/claude-sonnet-4-6": { alias: "Sonnet（快速）" },
        "openai/gpt-5.2": { alias: "GPT-5（強大）" },
        "google-antigravity/gemini-2-flash": { alias: "Gemini（備份）" },
      },
      imageModel: {
        primary: "anthropic/claude-sonnet-4-6",
        fallbacks: ["openai/gpt-5.2"],
      },
    },
  },

  // 提供者認證
  models: {
    providers: {
      anthropic: { apiKey: "${ANTHROPIC_API_KEY}" },
      openai: { apiKey: "${OPENAI_API_KEY}" },
      "google-antigravity": { apiKey: "${GEMINI_API_KEY}" },
    },
  },

  // 冷卻設定
  auth: {
    cooldowns: {
      billingBackoffHours: 5,
      billingBackoffHoursByProvider: {
        anthropic: 3,
        openai: 6,
      },
      billingMaxHours: 24,
      failureWindowHours: 24,
    },
  },
}
```

## 配置載入流程

```
1. 解析候選路徑
   - 環境變數 CLAWDBOT_CONFIG_PATH
   - Profile 檔案 ~/.openclaw/openclaw.{profile}.json
   - 預設檔案 ~/.openclaw/openclaw.json
       ↓
2. 讀取 JSON5 檔案
       ↓
3. 解析 $include 指令（遞迴載入）
       ↓
4. 環境變數替換（${VAR}）
       ↓
5. Zod Schema 驗證（未知鍵拒絕啟動）
       ↓
6. 應用預設值
       ↓
7. 路徑標準化
```

## 安全建議

1. **不要在配置中硬編碼敏感資訊** - 使用環境變數 `${VAR}` 或 Secrets 引用
2. **限制配置檔案權限** - `chmod 600 ~/.openclaw/openclaw.json`
3. **定期輪換 Token 和金鑰**
4. **`config get` 自動脫敏** - 敏感值在輸出中被遮蔽
5. **使用 SecretRef** - `tokenRef`/`keyRef` 引用取代 inline 敏感值
