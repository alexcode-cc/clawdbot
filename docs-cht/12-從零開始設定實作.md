# 從零開始設定實作指南

本指南將詳細說明如何從無到有設定 OpenClaw，實現以下完整工作流程：

1. **使用 Ollama 本地模型或 Claude Code 訂閱** 作為 AI 引擎
2. **使用 Discord 機器人** 作為通訊頻道
3. **自動收發 Gmail** 並觸發 AI 處理
4. **透過 Discord 機器人撰寫專案** 並自動上傳到 GitHub
5. **完成後發送 Gmail 通知** 並透過 Discord 告知

> **本指南預設使用 Ollama 伺服器（192.168.168.168:11434）作為 AI 引擎**，您也可以選擇使用 Claude Code 訂閱。

## 前置需求

### 系統需求

| 項目 | 需求 |
|------|------|
| 作業系統 | macOS 13.0+、Ubuntu 20.04+、Windows 10 1809+ |
| Node.js | >= 22.12.0 |
| RAM | 4 GB 以上 |
| 網路 | 需要穩定的網際網路連線 |

### 必要帳號

1. **Ollama 伺服器**（本地或遠端）或 **Claude 訂閱帳號**
2. **Discord 帳號**（用於建立機器人）
3. **Gmail 帳號**（用於自動收發郵件）
4. **GitHub 帳號**（用於程式碼託管）
5. **Google Cloud Platform 帳號**（用於 Gmail Pub/Sub）
6. **Tailscale 帳號**（用於公開 HTTPS 端點）

### 必要工具

```bash
node --version  # 應為 v22.x.x 或更高
npm install -g pnpm
git --version
```

## 第一步：安裝 OpenClaw

### 方法 A：使用 npm 全域安裝（推薦）

```bash
npm install -g openclaw@latest

# 驗證安裝
openclaw --version
```

### 方法 B：從原始碼建置

```bash
git clone https://github.com/openclaw/openclaw.git
cd openclaw

pnpm install
pnpm build

# 驗證
pnpm openclaw --version
```

### 初始設定

```bash
# 使用引導設定精靈
openclaw onboard --install-daemon
```

## 第二步：AI 模型設定

OpenClaw 支援多種 AI 模型提供者。以下說明 Claude Code 訂閱和本地 Ollama 伺服器的設定方式。

---

### 選項 A：使用本地 Ollama 伺服器（推薦用於本地部署）

#### A.1 確認 Ollama 伺服器運行中

```bash
curl http://192.168.168.168:11434/api/tags
```

#### A.2 設定 OpenClaw 使用遠端 Ollama

編輯 `~/.openclaw/openclaw.json`：

```json5
{
  models: {
    providers: {
      ollama: {
        baseUrl: "http://192.168.168.168:11434/v1",
        apiKey: "ollama-local",
        api: "openai-completions",
        models: [
          {
            id: "llama3.3",
            name: "Llama 3.3",
            reasoning: false,
            input: ["text"],
            cost: { input: 0, output: 0, cacheRead: 0, cacheWrite: 0 },
            contextWindow: 8192,
            maxTokens: 81920
          },
          {
            id: "qwen2.5-coder:32b",
            name: "Qwen 2.5 Coder 32B",
            reasoning: false,
            input: ["text"],
            cost: { input: 0, output: 0, cacheRead: 0, cacheWrite: 0 },
            contextWindow: 32768,
            maxTokens: 327680
          },
          {
            id: "deepseek-r1:32b",
            name: "DeepSeek R1 32B",
            reasoning: true,
            input: ["text"],
            cost: { input: 0, output: 0, cacheRead: 0, cacheWrite: 0 },
            contextWindow: 32768,
            maxTokens: 327680
          }
        ]
      }
    }
  },
  agents: {
    defaults: {
      model: {
        primary: "ollama/qwen2.5-coder:32b",
        fallbacks: ["ollama/llama3.3", "ollama/deepseek-r1:32b"]
      }
    }
  }
}
```

#### A.3 驗證 Ollama 設定

```bash
openclaw models list
```

---

### 選項 B：使用 Claude Code 訂閱

### B.1 安裝 Claude Code CLI

#### macOS / Linux / WSL

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

#### Windows PowerShell

```powershell
irm https://claude.ai/install.ps1 | iex
```

### B.2 登入並產生 Setup Token

```bash
claude
# 按照提示完成瀏覽器登入

claude setup-token
# 複製產生的 token
```

### B.3 在 OpenClaw 中設定 Claude

```bash
# 使用設定精靈
openclaw onboard --auth-choice setup-token

# 或使用命令列
openclaw models auth paste-token --provider anthropic
```

### B.4 驗證

```bash
openclaw models status
```

## 第三步：Discord 機器人設定

### 3.1 建立 Discord 應用程式

1. 前往 [Discord Developer Portal](https://discord.com/developers/applications)
2. 點擊 **New Application**
3. 輸入應用程式名稱（例如：`OpenClaw Assistant`）

### 3.2 建立 Bot 使用者

1. 在應用程式設定中，點擊左側 **Bot**
2. 點擊 **Add Bot**

### 3.3 取得 Bot Token

1. 在 Bot 頁面，點擊 **Reset Token** 或 **Copy**
2. **安全保存此 Token**

**警告：Bot Token 等同於密碼，絕對不要公開或提交到版本控制。**

### 3.4 啟用必要的 Gateway Intents

| Intent | 必要性 | 說明 |
|--------|--------|------|
| Message Content Intent | ✅ 必要 | 讀取訊息內容 |
| Server Members Intent | ✅ 建議 | 成員查詢、allowlist 比對 |

### 3.5 設定 OAuth2 權限並邀請 Bot

1. **OAuth2** → **URL Generator**
2. Scopes: `bot`, `applications.commands`
3. Bot Permissions: View Channels, Send Messages, Read Message History, Embed Links, Attach Files, Add Reactions
4. 複製 URL，在瀏覽器中開啟並邀請到伺服器

### 3.6 設定 OpenClaw Discord 配置

編輯 `~/.openclaw/openclaw.json`：

```json5
{
  channels: {
    discord: {
      enabled: true,
      token: "YOUR_DISCORD_BOT_TOKEN",
      
      dm: {
        enabled: true,
        policy: "allowlist",
        allowFrom: ["YOUR_USER_ID"]
      },
      
      guilds: {
        "YOUR_GUILD_ID": {
          slug: "my-server",
          requireMention: false,
          users: ["YOUR_USER_ID"],
          channels: {
            "YOUR_CHANNEL_ID": {
              allow: true,
              requireMention: false
            }
          }
        }
      },
      
      actions: {
        reactions: true,
        messages: true,
        threads: true
      }
    }
  }
}
```

## 第四步：Gmail 自動收發設定

### 4.1 使用 OpenClaw 設定精靈（推薦）

```bash
openclaw webhooks gmail setup \
  --account your-email@gmail.com
```

### 4.2 手動配置 Gmail

編輯 `~/.openclaw/openclaw.json`：

```json5
{
  hooks: {
    enabled: true,
    token: "YOUR_WEBHOOK_SECRET_TOKEN",
    path: "/hooks",
    presets: ["gmail"],
    
    gmail: {
      account: "your-email@gmail.com",
      label: "INBOX",
      topic: "projects/YOUR_PROJECT_ID/topics/gog-gmail-watch",
      subscription: "gog-gmail-watch-push",
      pushToken: "shared-push-token",
      hookUrl: "http://127.0.0.1:18789/hooks/gmail",
      includeBody: true,
      maxBytes: 20000,
      renewEveryMinutes: 720,
      
      serve: { bind: "127.0.0.1", port: 8788, path: "/" },
      tailscale: { mode: "funnel", path: "/gmail-pubsub" },
      
      model: "ollama/llama3.3",
      thinking: "off"
    },
    
    mappings: [
      {
        id: "gmail-to-discord",
        match: { path: "gmail" },
        action: "agent",
        wakeMode: "now",
        name: "Gmail Handler",
        sessionKey: "hook:gmail:{{messages[0].id}}",
        messageTemplate: "New email from {{messages[0].from}}: {{messages[0].subject}}",
        deliver: true,
        channel: "discord",
        to: "user:YOUR_DISCORD_USER_ID"
      }
    ]
  }
}
```

## 第六步：整合工作流程設定

### 6.1 完整配置檔

編輯 `~/.openclaw/openclaw.json`：

```json5
{
  // Ollama 提供者
  models: {
    providers: {
      ollama: {
        baseUrl: "http://192.168.168.168:11434/v1",
        apiKey: "ollama-local",
        api: "openai-completions",
        models: [
          {
            id: "qwen2.5-coder:32b",
            name: "Qwen 2.5 Coder 32B",
            reasoning: false,
            input: ["text"],
            cost: { input: 0, output: 0, cacheRead: 0, cacheWrite: 0 },
            contextWindow: 32768,
            maxTokens: 327680
          }
        ]
      }
    }
  },
  
  // Agent
  agents: {
    defaults: {
      model: {
        primary: "ollama/qwen2.5-coder:32b",
        fallbacks: ["ollama/llama3.3"]
      },
      tools: {
        exec: { enabled: true },
        browser: { enabled: true }
      }
    },
    list: [
      {
        id: "main",
        name: "OpenClaw",
        workspace: "~/Projects",
        identity: {
          name: "OpenClaw",
          soul: "你是一個專業的程式設計助手。你會使用繁體中文回應。"
        }
      }
    ]
  },
  
  // Discord
  channels: {
    discord: {
      enabled: true,
      token: "${DISCORD_BOT_TOKEN}",
      dm: {
        enabled: true,
        policy: "allowlist",
        allowFrom: ["YOUR_DISCORD_USER_ID"]
      },
      guilds: {
        "YOUR_GUILD_ID": {
          slug: "my-workspace",
          requireMention: false,
          users: ["YOUR_DISCORD_USER_ID"],
          channels: {
            "YOUR_CHANNEL_ID": { allow: true, requireMention: false }
          }
        }
      }
    }
  },
  
  // Gateway
  gateway: {
    mode: "local",
    port: 18789,
    bind: "loopback"
  },
  
  timezone: "Asia/Taipei",
  locale: "zh-TW"
}
```

### 6.2 環境變數檔案

建立 `~/.openclaw/.env`：

```bash
DISCORD_BOT_TOKEN=your-discord-bot-token
CLAWDBOT_HOOK_TOKEN=your-webhook-secret-token
GITHUB_TOKEN=ghp_your-github-token
```

## 第七步：啟動與測試

### 7.1 啟動 Gateway

```bash
# 啟動 OpenClaw Gateway
openclaw gateway run

# 或背景執行
nohup openclaw gateway run > /tmp/openclaw-gateway.log 2>&1 &
```

```bash
# 檢查整體狀態
openclaw doctor

# 檢查頻道狀態
openclaw channels status --probe

# 檢查模型認證
openclaw models status
```

### 7.3 測試 Discord 連線

在 Discord 中傳送訊息給機器人：

```
你好，請確認連線正常
```

在 Discord 中傳送：

```
請幫我建立一個簡單的 Python Hello World 專案，上傳到 GitHub，然後發送郵件通知我。
```

## 故障排除

### Discord 相關

```bash
# 檢查 Discord 連線狀態
openclaw channels status --probe

# 常見原因：
# 1. Message Content Intent 未啟用
# 2. Bot 權限不足
# 3. allowlist 設定錯誤
```

### Ollama 相關

```bash
# 確認伺服器運行中
curl http://192.168.168.168:11434/api/tags

# 確保 baseUrl 包含 /v1 後綴
# ✅ http://192.168.168.168:11434/v1
# ❌ http://192.168.168.168:11434
```

### Claude 相關

```bash
# 重新產生 setup-token
claude setup-token

# 重新設定
openclaw models auth paste-token --provider anthropic
```

### 一般診斷

```bash
# 完整診斷
openclaw doctor

# 查看日誌
tail -f /tmp/openclaw-gateway.log

# 重啟 Gateway
openclaw gateway run --force
```
