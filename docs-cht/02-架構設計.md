## 核心設計模式

### 1. 依賴注入模式

專案使用多種依賴注入方式，確保模組可測試和可替換：

#### 函數式 DI（CLI 層）

```typescript
// src/cli/deps.ts
export type CliDeps = {
  sendMessageWhatsApp: typeof sendMessageWhatsApp;
  sendMessageTelegram: typeof sendMessageTelegram;
  sendMessageDiscord: typeof sendMessageDiscord;
  // ... 更多依賴
};

export function createDefaultDeps(): CliDeps {
  return {
    sendMessageWhatsApp,
    sendMessageTelegram,
    sendMessageDiscord,
    // ...
  };
}

// 使用方式
const deps = createDefaultDeps();
await deps.sendMessageTelegram(chatId, message);
```

#### 工廠函數 DI（Config 層）

```typescript
// src/config/io.ts
export type ConfigIoDeps = {
  fs?: typeof fsPromises;
  logger?: Logger;
  env?: Record<string, string | undefined>;
};

export function createConfigIO(overrides: ConfigIoDeps = {}) {
  const deps = {
    fs: overrides.fs ?? fsPromises,
    logger: overrides.logger ?? defaultLogger,
    env: overrides.env ?? process.env,
  };

  return {
    loadConfig: () => loadConfigImpl(deps),
    saveConfig: (config) => saveConfigImpl(deps, config),
  };
}
```

#### Options 物件 DI（Agent 層）

```typescript
// src/agents/openclaw-tools.ts

  const opts = { ...defaultOptions, ...options };
  // 使用 opts 建立工具
}
```

## 安全考量

1. **執行沙箱**: 工具執行受限於特定目錄
2. **審批機制**: 敏感操作需使用者確認（exec approvals）
3. **認證**: Gateway 支援 Token 和設備身份認證
4. **加密**: 支援 TLS (WSS) 和端對端加密頻道
5. **安全掃描器** (`src/security/skill-scanner.ts`): 自動掃描技能/插件程式碼
   - 偵測規則：危險執行 (child_process)、動態程式碼 (eval)、加密挖礦、可疑網路連線、資料外洩、程式碼混淆、環境變數蒐集
   - 嚴重等級：`critical`、`warn`、`info`
6. **命令授權分離**: `commands.allowFrom` 可獨立限制命令使用者
7. **Discord 元件安全**: Agent Components 強制 DM 授權檢查、防止頻道 ID 偽造（支援 Components v2 cid）
8. **會話自動維護**: 自動清理過期條目、限制儲存大小、檔案輪轉
9. **配置值脫敏**: `config get` 輸出自動遮蔽敏感值
10. **安全二進位政策**: `exec-safe-bin-policy` 控制可執行的二進位檔
11. **沙箱繼承強化**: `sessions_spawn` 禁止沙箱降級跨代理生成
12. **TOCTOU 防護**: `writeFileWithinRoot` 防止 symlink 競爭條件
13. **節點元資料策略**: Unicode 混淆防護、canonical 路徑驗證
14. **Secrets 管理**: SecretRef 支援 64 個憑證目標，涵蓋完整使用者提供的憑證表面
15. **WS 安全策略**: 預設僅允許 loopback `ws://`，私網需明確 opt-in
16. **HTTP 安全標頭**: `Permissions-Policy` + `X-Content-Type-Options: nosniff`
17. **懶加載邊界**: 專用 `.runtime.ts` 邊界防止混合靜態/動態 import，建置時檢查 `[INEFFECTIVE_DYNAMIC_IMPORT]` 警告
18. **Plugin Prompt Hook 策略**: 外掛 prompt hooks 加入 runtime policy 驗證
19. **Context Engine 安全**: context-engine 外掛透過 `api.runtime.modelAuth` 存取 model auth API
20. **Cron Store 權限**: cron store 和 run logs 強制 `0600` 權限（owner-only）
21. **Exec Fail-Open 修復**: exec script preflight 中的 fail-open 繞過已關閉
22. **Browser SSRF 防護**: 透過即時路由攔截封鎖 SSRF redirect 繞過、CDP URL/endpoint/websocket 全面驗證
23. **Sandbox Home Credential Binds**: 封鎖 home credential bind mounts
24. **Bootstrap Token 撤銷**: handshake 完成後立即撤銷 bootstrap tokens
25. **設備配對 Scope 強化**: 端到端配對 approval scope 強制執行、設備管理授權拒絕未核准角色
26. **Prompt Cache 穩定性**: MCP 工具確定性排序、壓縮策略保護 cache prefix、transport 分離
27. **Hook Fail-Closed**: `before_tool_call` hook runner 錯誤時 fail-closed
28. **Memory Exhaustion 防護**: 內聯圖片解碼記憶體耗盡防護
29. **Context Leak 修復**: 關閉剩餘 internal-context 洩漏路徑
30. **Untrusted File Wrapping**: 不信任的檔案輸入包裝保護
31. **SSRF 防護增強（2026.4.8）**: Slack proxy 支援、DNS pinning skip for trusted proxy、SSRF redirect guard 加強、HTTP/1.1 SSRF 分發器集中化
32. **NO_PROXY 語義修正（2026.4.8）**: leading-dot NO_PROXY 條目和 malformed URL 正確處理
33. **Env Blocklist 擴充（2026.4.8）**: Java、Rust、Cargo 工具鏈敏感環境變數封鎖
34. **Secret Rotation Session 失效（2026.4.8）**: shared-token/password WS sessions 在 rotation 後立即失效
35. **Node Reconnect Re-pairing（2026.4.8）**: node 重連 command 升級需重新配對
36. **Base64 Decode Guard（2026.4.8）**: 遺漏的 base64 解碼路徑安全防護
37. **Commentary History Leak Fix（2026.4.8）**: 修復 commentary 歷史記錄洩漏
38. **Cross-origin Redirect Body Drop（2026.4.8）**: 跨域 unsafe-method 重定向移除 request body
39. **Busybox/Toybox 移除（2026.4.12）**: exec 安全策略移除 busybox/toybox 以防止命令別名繞過
40. **Shell-wrapper 偵測擴充（2026.4.12）**: 擴展 shell-wrapper 偵測範圍，涵蓋更多間接執行路徑
41. **Empty Approver List 修復（2026.4.12）**: 修復空核准者列表導致的 fail-open 問題
42. **OAuth URL Injection 修復（2026.4.12）**: 防止 OAuth callback URL 注入攻擊
